Current: D→K=16→D. The bilinear step has K² = 256 off-diagonal entries. Increasing to K=32 gives 1024 entries — 4× more cross-dim mixing at cost of 2× bind params (negligible).

2.3 Revive Spectral via Initialization

lambda_k is frozen near init because the DCT residual path has no learnable gain to compete with the main path. With h = h + (h_dct * lambda_k) @ V, the spectral path output is O(lambda_k * ||h||) ≈ O(0.5 * 1.0) ≈ 0.5, which is small relative to the main path's signal.

Fix: Initialize lambda_k with larger spread: torch.randn(cfg.D) * 0.1 + 0.5. Or: add a learned output gate for the spectral path: h = h + spectral_out * sigmoid(g).

2.4 VSA Gates: ReLU Decay + Learned Bias

Current decay is sigmoid(h·w_d + b_d) which saturates at [0, 1]. With b_d ≈ 0, the decay is ~0.5 regardless of input.

Option A: Initialize b_d = 2.0 (start high, decay slowly). Let training pull it down.

Option B: Replace sigmoid with min(1, exp(h·w_d + b_d)) for unbounded decay (can be >1 for "forget more" or ≈1 for "keep").

Mathematical note: With decay ≈ 0.5, the effective context window is roughly τ = 1/(1-0.5) = 2 steps. With L=128, this means the memory sees only ~2 tokens of context. To reach τ = 128, we need decay > 0.992. This is why VSA state appears critical for loss < 9 — even 2-step memory is vastly better than no memory (random). But there's room for much longer memory.

2.5 Cognitive Mirror — Replaces Old Mirror Bind

  • Smooth path: h[t] — conv1×3(h[t]) (local coherence in K-space)
  • Symmetry path: (h[t] ⊙ wᵤ) · (h[t-1] ⊙ wᵥ) (bilinear trajectory)
  • All three → rms_norm → tanh(W_out) → bounded correction ±exp(log_scale)
  • log_scale per-dim gate (896 dims, init=0, exp=1) gets full gradient signal
Gradient verification: log_scale gradient non-zero on all 896 dims. Output bounded to |mirror| ≤ exp(log_scale). Contrast with old mirror: unbounded (max=5.05), scalar scale (gradient 0.1× weaker).

Part 3: Mathematical Verification

3.1 Effective Rank of Bind

Claim: The bind path D→K→D with bilinear in K can represent any rank-≤K matrix.

Proof: The bilinear operation bind(h) = ((h·W_p) ⊙ w_u ⊙ w_v) · W_o can be rewritten as:

bind(h) = h · W_p · diag(w_u) · diag(w_v) · W_o
        = h · (W_p · D · W_o)

where D = diag(w_u) · diag(w_v) is a rank-K diagonal matrix in the bottleneck space.

This gives bind(h) = h · M where M = W_p · D · W_o is a D×D matrix with rank ≤ K. With K=16, the bind can represent any rank-16 perturbation of the residual stream. For W_p and W_o to be full-rank on their own (rank K), both must have rank K, which requires inputs/outputs to not lie in a lower-dimensional subspace. The eff_rank(W_p) ≈ 12.8 (out of 16) confirms the bind is using ~80% of its theoretical capacity.

Implication: Increasing K to 32 would give rank-32 binding at negligible cost (4K extra params per layer). The bind is not the bottleneck — it's the MLP at 1× expansion.

If the input has intrinsic dimension d < 896 (which is likely for language after 24 layers), the MLP compresses and decompresses through the same 896-dim space, losing information. The rank-228 observation suggests the data manifold has intrinsic dimension ~228 at this representation level.

Observation: RMS |grad| = 7.9×10⁻⁵, RMS grad² = 1×10⁻⁶. For AdamW (β₁=0.9, β₂=0.95, ε=1×10⁻⁸):

Effective ΔW per step ≈ 6.7×10⁻⁵. Over 20K steps, cumulative change ≈ 1.34. Since |W| ≈ 28 (early layers) to 77 (last layer), the relative change is ~2-5% — reasonable for 20K steps. The learning is happening, but slowly.

Prediction: Given 500K total steps, the model will have ~33% cumulative weight change in early layers and ~5% in late layers. The last-layer collapse will persist or worsen. Spectral and mirror paths will remain frozen.

3.4 VSA Memory Time Constants

VSA memory recurrence: mem[t] = decay[t] · mem[t-1] + input[t] · i_gate[t]

With decay ≈ 0.5 (constant), the memory after k steps is:

mem[t] = Σ_{j=0}^{k-1} 0.5ʲ · input[t-j] · i_gate[t-j] + 0.5ᵏ · mem[t-k]

The effective time constant τ = 1/(1-0.5) = 2 steps. This means only the last 2 tokens contribute significantly to the memory state. After 10 steps, the contribution is 0.5¹⁰ ≈ 0.001 — effectively zero.

To get τ = 128 (the sequence length):

τ = 1/(1-α) → α = 1 - 1/128 = 0.9922

With b_d initialized to 0 and sigmoid activation, α ≈ 0.5. To get α = 0.992, we need b_d ≈ 4.8 (since sigmoid(4.8) ≈ 0.992).

Fix: Initialize b_d = 5.0 (decay ≈ 0.993) and let training adjust. This immediately gives τ ≈ 150 — the memory can span the full 128-token sequence from the start of training.

Caveat: With b_d = 5.0 and w_d init std=0.1, the input-dependent modulation is ≈ 5 ± 0.1·std(h). Since std(h) ≈ 1.0, the decay varies in [0.991, 0.995] — a narrow range. To make it content-dependent, increase w_d std to ~1.0.

3.5 DCT Orthogonality Error Propagation

The non-orthogonal DCT basis introduces aliasing. With ||VVᵀ - I|| = 1.1171, the error per spectral transform step is ~1.12·std(h) ≈ 1.12. Since the spectral path is additive (h = h + spectral_out), the error compound is:

h^{(l+1)} = h^{(l)} + ((h^{(l)}·Vᵀ) ⊙ λ) · V
          ≈ h^{(l)} + error_term(h^{(l)}) + ideal_term(h^{(l)})

After 24 layers, the total aliasing error could be ~24 × 1.12 ≈ 27 relative to signal std=1. This is likely compensated by the learned lambda_k (which stays near 0.5, reducing spectral path gain), but it means the spectral path is operating at ~50% efficiency.

Fix the bug and the basis becomes orthogonal (error ≈ 10⁻⁶), enabling lambda_k to learn meaningful frequency selective filters.

3.6 Zeckendorf Embedding Capacity

Zeckendorf codes: vocab = 50000 tokens, K = 23 bits/token. The code matrix Z ∈ {0,1}^{50000×23} has the property that each row has sum ≈ 6.19 (average number of Fibonacci terms needed).

Capacity: The linear projection W_p ∈ ℝ^{23×896} maps these codes to D-space. The Gram matrix of codes: G = Z·Zᵀ. If two codes are similar (share Fibonacci terms), their representations are correlated. The maximum possible distinct representations is bounded by the rank of G.

Problem: Zeckendorf codes are sparse with fixed support — they provide a very specific geometry in {0,1}²³ that may not match the natural geometry of the D-space. Making the projection learnable (full 50000×896 lookup table) would add 44.8M params — not feasible.

Better: Add a small learned delta on top of the fixed Zeckendorf projection: embed[t] = Z[t] · W_p + E[t] where E ∈ ℝ^{50000×D} is a learned embedding matrix initialized at 0. This adds 44.8M params (too much for MX550). Alternative: factorized embedding — learn E = Z · W_e where W_e ∈ ℝ^{23×896} is learnable (only 20K extra params). Wait, that's already what we have! W_p is already and solely responsible for the mapping from binary codes to D-space. So the current design is already a compressed embedding via Zeckendorf codes.

The question is whether the Zeckendorf code structure is optimal. Zeckendorf codes have the property that no two consecutive bits are both set — this ensures unique representation. But for neural embeddings, this sparsity may not be helpful. Removing the Zeckendorf constraint and just using a learned K=23 embedding per token (50000×23 = 1.15M params) would be more flexible and still far smaller than a full 50000×896 embedding.
