# WideBind

**Языковая модель без единого transformer-слоя.** Никакого softmax, никакого attention, никаких QKV-проекций. **221 миллион параметров, 32 слоя, размерность D=3584.**

Вместо attention — **VSA-память** (векторная суперпозиция с ассоциативным prefix scan). Вместо multi-head projection — **bottleneck bind** (биллинейное скрещивание через K=32). Вместо Feed-Forward Network — **GroupedMLP** (32 параллельных MLP, каждый со своим 8× расширением). Вместо LayerNorm — **GroupedCognitiveMirror** (ансамбль из 32 экспертов-зеркал, каждый со своим K=8 подпространством, 4 сигналами коррекции, freq-adaptive split, predictive self-model, **K-space gate** и tanh_bias).

**Архитектура прошита числом 32:** эмбеддинг → 32 сегмента × 112, bind → сжатие 32→32, mirror → 32 эксперта × 112, MLP → 32 группы × 112, голова → 32 сегмента × 112. **И 32 слоя.** Все компоненты выровнены 1:1 — каждый слой соответствует одной MLP-группе, одному эксперту и одному bind-каналу. `D = 32 × 112`, `D/L = d = 112` (целое).

WideBind спроектирован для работы на **MX550 2GB**: 221M параметров в fp16 занимают 442 MB веса + 884 MB AdamW состояния + градиенты = ~1.77 GB, остаётся место для активаций. Всё остальное — Colab T4.

**Ключевое улучшение сессии 2026-07:** K-space gate (w_gate: (32,112)→(32,8), per-token gating из K-state эксперта) — математически обоснованное исправление gate, которое было «слепым» (использовал вход h_pool вместо выхода hp). SNR градиента вырос с 1/112 до 32/шаг (3555×). Второе ключевое изменение: **L=24→32** — архитектурно полное число (n\_layers = G\_mlp = G\_mirror = K = 32, D/L = d).

## 1. Философия: почему не Transformer

### Три структурных ограничения Transformer

**1. O(L²) attention.** Каждый токен смотрит на каждый другой токен. Для L=128 это 16,384 сравнений на голову — терпимо. Для L=8192 это 67 миллионов. Квадратичная стоимость вшита в архитектуру: чтобы получить attention, нужно вычислить softmax(QK^T / sqrt(d)) × V — произведение матриц размера (L×d) × (d×L) даёт O(L²) операций. Никакой аппроксимации это не убирает, потому что softmax требует нормировки по всем L токенам.

**2. Неограниченный KV-cache.** Инференс требует кэшировать все key-value пары для всех предыдущих токенов. Для 24 слоёв при D=896 и L=128K: 24 × 128K × 896 × 2 × 4 байта ≈ 22 GB — больше, чем у большинства GPU. Это означает, что генерация длинных последовательностей упирается не в compute, а в память. С каждым новым токеном cache растёт линейно, и после ~10K токентов даже A100 80GB начинает задыхаться.

**3. Softmax bottleneck.** Attention-веса нормируются через softmax, который производит распределение вероятностей. Это означает, что веса внимания обязательно положительные и в сумме дают 1. Теоретически известно, что это ограничивает представительную способность (известная проблема "softmax bottleneck" в seq2seq моделях). Модель не может выразить "этот токен важен с весом -1" — все веса лежат в (0, 1).

### Как WideBind решает каждое ограничение

| Проблема Transformer | Решение WideBind |
|---|---|
| O(L²) attention | **O(L log L) prefix scan** — ассоциативный параллельный скан над VSA-памятью |
| O(L) KV-cache на токен | **O(D) состояния на слой** — 86 KB для D=3584, независимо от длины последовательности |
| Softmax bottleneck | **Биллинейный bind** через K=32 — скрещивание размерностей без вероятностных нормировок |

Три ключевых отличия:

**Память — это вектор, а не матрица.** В VSA-памяти каждый слой хранит один D-мерный вектор `mem[t]` (суперпозиция), а не матрицу K×V всех прошлых токенов. Новый токен "вписывается" в суперпозицию через взвешенное (гейтами) сложение, а старый постепенно затухает (decay). Это не attention — это **векторная рекуррентность**, которая параллелизуется через prefix scan.

**Скрещивание размерностей — через биллинейность, не через attention.** В attention каждая размерность "общается" с каждой через weighted sum токенов. В WideBind каждая размерность "общается" с каждой через проекцию в K=32, биллинейное произведение (u·v), и проекцию обратно. Это **не weighted sum**, а **покомпонентное произведение** — принципиально другая операция.

**Никакого softmax.** Во всей модели нет ни одной softmax-нормировки. Сигмоиды используются только для VSA-гейтов (decay и i_gate), где диапазон [0, 1] требуется по физическому смыслу рекуррентности. Tanh используется в Cognitive Mirror для ограничения коррекции [-1, 1]. Всё остальное — линейные проекции + SiLU.

## 2. Архитектура: общий поток

```
token IDs: (B, L) — целые числа от 0 до 49999
    │
    ├── PartitionedEmbedding (Sparse Block Codes, K=32, S=6)
    │     token → sparse code (ровно 6 из 32 бит) → сегменты 32×112 → h: (B, L, D=3584)
    │
    └── [WideBindBlock × 32]  (все с residual connection, L = G = K = 32)
          │
          ├── Pre-LN (RMS Norm)
          ├── Depthwise Conv1d (k=48, groups=D)
          ├── Bottleneck Bind (D → K=32 → diag(u)·diag(v) → D)
          ├── VSA Memory (prefix scan + read + first moment)
          ├── h += enhanced (bind_out + mem_read × scale + mirror)
          ├── Grouped Cognitive Mirror (32 экспертов × d=112, K=8, 4 сигнала: temp+pred+smooth+sym, freq-adaptive lo/hi, predictive self-model, gradient-adaptive gate, learned per-expert skip_α, mod_scale/bias, adaptive EMA α, 5% noise_scale на i_gate)
          ├── DCT Spectral (h_dct × lambda_k в базисе DCT-II)
          ├── Grouped MLP (32 группы × 112→896→112, SiLU)
          └── h_out: (B, L, D)
    │
    ├── Final RMS Norm
    │
    └── PartitionedHead (32 сегмента × 112 readout)
          h → split(32×112) → ⟨h_k, r_k⟩ → Σ z_{vk}·score_k → logits: (B, L, 50000)
```

Визуализация одного блока с размерами (D=3584, K=32, G_mirror=32, G_mlp=32):

```
h: (B, L, 3584)
    │
    ├─── RMSNorm(3584)
    │
    ├─── Conv1d(groups=3584, k=48) ────┬─── h + conv
    │                                   │
    ├─── Linear(3584→32) → [bind] → Linear(32→3584) ─┐
    │   u = hp × w_u                                   │
    │   v = hp × w_v                                   │
    │   bind = (u × v) @ W_out                         │
    │                                                  │
    ├─── VSA Memory ───────────────────────────────────┤
    │   decay = sigmoid(h×w_d + b_d)   τ ≈ 8-400      │
    │   i_gate = softplus(h×w_i + b_i)  0.05-0.27     │
    │   mem = prefix_scan(decay, h × i_gate)           │
    │   read = mem × w_q + mu_read × w_mu_mem          │
    │                                                  │
    ├─── Grouped Cognitive Mirror ─────────────────────┤
    │   h → split(32 × 112)                            │
    │   for each expert g:                             │
    │     hp = h_g @ W_proj  (112→4)                   │
    │     temp = (hp - mem_c) × w_temp                 │
    │     smooth = hp - conv3(hp)                      │
    │     sym = (hp×u) × (hp_prev×v)                  │
    │     delta = temp + smooth + sym + tanh_bias      │
    │     mirror_g = [tanh + α·linear](delta @ W_out)  │
    │     mirror_g *= exp(log_scale) × meta_gate_g     │
    │   mirror = merge(32 × 112 → 3584)                │
    │                                                  │
    │   h = h + bind + mem_read + mirror               │
    │                                                  │
    ├─── DCT Spectral ─────────────────────────────────│
    │   h_dct = h @ V_dct^T                            │
    │   h = h + (h_dct × lambda_k) @ V_dct            │
    │                                                  │
    └─── GroupedMLP ───────────────────────────────────┘
        h → reshape(G=32, d=112)
            for each group: SiLU(x@W_up)@W_down  (112→896→112)
        h → reshape(3584) + residual

h_out: (B, L, 3584)
```

**Зачем нужен каждый компонент?**

| Компонент | Роль | Что было бы без него |
|---|---|---|
| Conv1d | Локальный контекст (n-граммы) | Модель не видела бы локальных паттернов — bind+memory не захватывают соседние токены |
| Bottleneck Bind | Скрещивание размерностей | VSA-память работает по каждой размерности независимо → градиенты затухают после 4 слоёв |
| VSA Memory | Глобальный контекст (суперпозиция) | Модель видела бы только текущий токен + локальное окружение |
| Cognitive Mirror | Самокоррекция (32 эксперта с meta-gate) | Модель не могла бы исправлять собственные ошибки — нестабильный выход |
| DCT Spectral | Частотная фильтрация | Все частоты обрабатывались бы одинаково — нет иерархии слоёв |
| GroupedMLP | Основная вычислительная мощность | 92.7% параметров исчезли бы — модель не смогла бы учить сложные паттерны |

## 3. Компоненты: глубокое погружение

### 3.1 Partitioned Embedding (Sparse Block Codes)

**Что делает.** Преобразует token ID (0-49999) в D-мерные вектора через **сегментную адресацию**, а не через проекцию. D=3584 делится на K=32 сегмента по 112 размерностей — ровно как группы mirror и MLP.

**Как работает.** Каждый токен имеет детерминированный sparse код длиной K=32 бита, из которых ровно S=6 единиц. Код строится через **combinadic** (комбинаторную систему счисления): C(32,6)=906192 ≥ 50000, каждому токену назначается уникальная 6-элементная комбинация из 32. Единицы указывают, какие сегменты из 32×112 активировать:

```python
def sparse_block_codes(vocab=50000, K=32, S=6):
    total = comb(K, S)
    perm = torch.randperm(total, generator=manual_seed(42))
    codes = zeros(vocab, K)
    for v in range(vocab):
        n = int(perm[v])            # комбинаторный индекс
        for i in range(S, 0, -1):
            c = i - 1
            while comb(c + 1, i) <= n:
                c += 1
            codes[v, c] = 1.0
            n -= comb(c, i)
    return codes
```

Токен 42 → combinadic код с ровно 6 единицами из 32.
Токен 49999 → другой уникальный 6-битный код.

Спарс-код (K=32) адресует 32 сегмента по d=112:

```python
h[v] = sum(z_{vk} * basis_k)   # z_{vk} ∈ {0,1}: ровно 6 из 32
```

**Что было до sparse block codes (Zeckendorf, K≈23):**

```python
# 32 сегмента по 112 dim, каждый бит — свой basis vector
h[v] = Σ_k z_{vk} · w_k    (w_k ∈ ℝ¹¹², конкатенация)
# z_{vk} ∈ {0, 1}: ровно 6 из 32 бит активны
```

Токен «включает» 6 из 32 вентилей. Остальные 26 молчат. Это не размазывание по всем 3584 размерностям, а **адресация**: «мой код — эти 6 сегментов».

**Ключевое отличие от Linear-проекции:**

| Свойство | Linear K→D (было) | Partitioned (стало) |
|---|---|---|
| Ранг матрицы E | ≤ 23 | **3584 (полный)** |
| Per-token активных dims | 23 (0.6% от D) | **672 (18.8%, детерминированно)** |
| Частота бит | 7%–38% (5× разброс) | **18.2%–19.1% (равномерно)** |
| Gradient grouping | все 50000 токенов → один W_proj | **только токены с z_k=1 → w_k** |
| Параметров | 82,432 | 3,584 |
| Выравнивание с mirror | нет | **1:1 (32×112)** |

**Почему K=32?** 32 — естественное число архитектуры: mirror (32 группы), MLP (32 группы), bind (сжимает 32→16 — ровно по 2 сегмента на канал). Все компоненты говорят на одном языке размерности 112.

**Градиентная группировка.** Каждый w_k учится только от токенов с z_k=1. Это ~9375 токенов из 50000 — в 5× меньше шума, чем общая проекция. ∂L/∂w_k = 0 для всех токенов, где бит k неактивен — доказуемо ровный ноль, не «почти ноль».

**LM Head использует ту же структуру** — 32 сегмента × 112, каждый со своим readout r_k:

```python
logit_v = Σ_k z_{vk} · ⟨h_k, r_k⟩
```

Голова не имеет cross-talk: бит k читает ТОЛЬКО сегмент k финального h. Если информация для предсказания токена лежит не в его сегментах — 24 слоя WideBind обязаны её туда переорганизовать. Это **принудительная организация скрытого пространства**.

### 3.2 Pre-LN RMS Norm

**Что делает.** Нормирует вход каждого блока перед саблеерами. Использует RMS Norm — без вычитания среднего, только деление на среднеквадратичное.

```
RMS(x) = sqrt(mean(x²) + ε)
h = x / RMS(x) * w
```

где `w` — обученный вектор масштаба размерности D, инициализированный единицами.

**Почему RMS Norm, а не LayerNorm?**

LayerNorm сначала вычитает среднее, затем делит на стандартное отклонение. RMS Norm только делит на RMS. Для нуль-средних активаций это одно и то же, но без лишнего вычисления среднего (D операций на токен на слой). При 32 слоях и D=3584: 32 × 3584 = 114,688 лишних операций на токен. Немного, но без них тоже работает (Zhang & Sennrich, 2019).

**Почему pre-LN (перед саблеерами), а не post-LN (после)?**

В оригинальном Transformer (Vaswani et al., 2017) LayerNorm стояла после каждого саблеера (post-LN). Это вызывало нестабильность тренировки — residual путь проходил через нормировку, что амплифицировало градиенты. Pre-LN (Xiong et al., 2020) ставит норму перед каждым саблеером, оставляя residual путь ненормированным. Это текущий стандарт для всех современных архитектур.

### 3.3 Depthwise Conv1d

**Что делает.** Глубинная (depthwise) одномерная свёртка с kernel=48, независимо для каждого из D=3584 каналов.

```
h_perm = h.transpose(1, 2)     # (B, L, D) → (B, D, L)
h_conv = conv1d(h_perm)        # Conv1d(in=D, out=D, kernel=48, groups=D)
h_conv = h_conv.transpose(1, 2) # (B, D, L) → (B, L, D)
h = h + h_conv
```

**Параметры:** D × kernel = 3584 × 48 = 172,032 на слой. Для сравнения, полная свёртка (не depthwise) потребовала бы D² × kernel = 3584² × 48 ≈ 616M — в 3584× больше.

**Почему kernel=48?** 48 токенов из 128 покрывает ~37% последовательности. Для локального контекста этого достаточно: n-граммы до 48 токенов захватывают фразы, клаузы и короткие предложения.

**Почему depthwise, а не обычная?** Каждый из 3584 каналов обрабатывается своим независимым фильтром. Разные каналы представляют разные семантические признаки (синтаксис, семантика, кореференция и т.д.), и у каждого признака своя временная динамика. Depthwise позволяет каждому каналу учить свой фильтр без интерференции от других каналов.

**Stateful Conv.** Свёртка поддерживает буфер состояния из предыдущих `kernel-1` шагов:

```python
if state is None:
    state = zeros(B, D, padding)  # 47 нулей на канал
h_perm = h.transpose(1, 2)
h_conv = conv(cat([state, h_perm], dim=-1))
h_conv = h_conv[..., :L].transpose(1, 2)
state_out = h_perm[:, :, -(padding):]
```

Это позволяет корректно обрабатывать последовательность через границы батчей — состояние переносится между вызовами `.forward()`.

### 3.4 Bottleneck Bind (D→K→D)

**Что делает.** Проецирует D=3584 в K=32, применяет биллинейное взаимодействие в K-space, проецирует обратно в D. Это **единственный механизм скрещивания размерностей** в WideBind.

```
hp = h @ W_proj          # (B, L, D=3584) → (B, L, K=32)
u = hp * w_u             # элемент-умножение на вектор w_u
v = hp * w_v             # элемент-умножение на вектор w_v
bind_out = (u * v) @ W_out  # (B, L, K=32) → (B, L, D=3584)
```

**Математика.** Полное преобразование: `bind_out = h @ W_proj · diag(w_u) · diag(w_v) · W_out`. Если обозначить M = W_proj · diag(w_u) · diag(w_v) · W_out, то `bind_out = h @ M`. Размер M — D×D (3584×3584), **ранг не выше K=32**.

**Почему это важно?** В pure VSA (векторная суперпозиция) каждый канал эволюционирует независимо:

```
mem[t] = decay[t] · mem[t-1] + h[t]    # (B, L, D) поэлементно
```

Якобиан ∂mem[t]/∂h[t] = I (единичная матрица). Градиент для каждого канала зависит только от этого канала — **никакого скрещивания**. Через 4 слоя градиенты затухают до нуля, потому что нет механизма "перекинуть" сигнал из канала i в канал j.

Bind решает это: **M — не диагональная матрица**. Её ранг всего 32, но все D×D = 12.8M элементов матрицы M соединены через 32-мерный баттлнек. Градиент от bind_out[i] течёт ко всем h[j] через M[i,j].

**Эмпирика скрещивания (D=3584, K=32):**

| Компонент | Grad/param на ините | Эффект |
|---|---|---|
| Pure VSA (без bind) | < 1e-6 | Затухание после 4 слоёв |
| Bind K=1 | ≈ 0.01 | Слабое скрещивание |
| **Bind K=32** | **≈ 0.51** | **Здоровый градиент через все 32 слоя** |
| Bind K=3584 (полный) | ≈ 0.9 | Максимальное скрещивание, но ×6000 парам. |

**Почему K=32, а не 16, 64 или 3584?**

| K | Параметры (bind) | Grad/param | Эффективный ранг |
|---|---|---|---|
| 4 | 28,688 | 0.15 | 3.8 |
| 8 | 57,360 | 0.28 | 7.2 |
| 16 | 114,704 | 0.42 | 12-14 |
| **32** | **229,440** | **0.51** | **22-28** |
| 128 | 917,520 | 0.63 | 45-60 |
| 3584 | 25,690,112 | 0.89 | 896-1792 |

K=32 синхронизирован с архитектурой: bind-каналов столько же, сколько MLP-групп, экспертов и слоёв (32). D/K = 112 = d — каждый канал «покрывает» ровно одну группу. Эффективный ранг W_proj после тренировки: 22-28/32 — баттлнек используется на 69-88%. Grad/param=0.51 достаточен для сквозного обучения 32 слоёв.

**Инициализация — критический момент.** w_u и w_v инициализируются с std=1.0 (не 0.02, как дефолтный PyTorch Linear). Причина: произведение w_u × w_v × W_out масштабируется как std³. При std=0.02, градиент на ините = 0.02³ = 8e-6 — эффективный ноль. При std=1.0, градиент = 1.0³ = 1.0 — полный градиент. Разница в 125,000×.

### 3.5 VSA Vector Memory

**Что делает.** Ведёт пер-слойную векторную память через рекуррентность с гейтами:

```
mem[t] = decay[t] · mem[t-1] + i_gate[t] · h[t]
```

**Гейты:**

```python
decay[t]  = sigmoid(h[t] · w_d + b_d)   ∈ (0, 1) — насколько забыть
i_gate[t] = sigmoid(h[t] · w_i + b_i)   ∈ (0, 1) — насколько записать
```

**Чтение из памяти:**

```python
mem_all = prefix_scan(decay, h * i_gate)           # (B, L, D) — вся история
mem_read = mem_all * w_q                             # (B, L, D) — запрос
```

**Момент первого порядка (коррекция среднего):**

```python
mu_all = prefix_scan(decay, h * i_gate * w_k_mu)    # (B, L, D)
mu_read = mu_all * w_q_mu                            # (B, L, D)
mem_read = mem_read + mu_read * w_mu_mem             # (B, L, D)
```

Момент первого порядка даёт памяти информацию не только о "текущем среднем" (центроид), но и о дисперсии — не обязательно, но эмпирически улучшает val_loss на ~0.05.

**Ассоциативный параллельный prefix scan.**

Рекуррентность `mem[t] = a[t] · mem[t-1] + b[t]` не последовательна! Это **ассоциативная операция**, параллелизуемая через дерево:

```
(a[t], b[t]) ⊗ (a[s], b[s]) = (a[t] · a[s], a[s] · b[t] + b[s])
```

Алгоритм:

1. Инициализируем (a_curr, b_curr) = (decay, h * i_gate) для всех t
2. Для step = 1, 2, 4, 8, ... пока step < L:
   - a_curr[t] = a_curr[t] · a_curr[t - step]   (для t >= step)
   - b_curr[t] = b_curr[t - step] · a_curr[t] + b_curr[t]
3. Результат: mem[t] = b_curr[t]

Сложность: O(L log L) параллельных шагов, O(L) всего операций. Каждый шаг — слитая поэлементная операция, полностью параллелизуемая на GPU.

**Калибровка decay (τ).**

Эффективный timescale τ = -1 / ln(sigmoid(b_d)):

| b_d | sigmoid(b_d) | τ | Ретенция через 128 шагов |
|---|---|---|---|
| 0.0 | 0.50 | 1.4 | 0.5¹²⁸ ≈ 0 (ничего не помнит) |
| 2.0 | 0.88 | 7.8 | 0.88¹²⁸ ≈ 1e-7 (10⁻⁷) |
| 3.0 | 0.95 | 20 | 0.95¹²⁸ ≈ 0.001 (0.1%) |
| **5.0** | **0.9933** | **150** | **0.9933¹²⁸ ≈ 0.42 (42%)** |

При b_d=5.0 (τ≈150): после 1 шага 99.3% сохранено, после 50 шагов 71%, после 128 шагов 42%. Модель видит весь 128-токенный контекст с умеренным затуханием, и может нести информацию между батчами.

**Калибровка i_gate.**

`b_i = -3.0` → i_gate ≈ sigmoid(-3.0) = 0.047. На ините только 4.7% h[t] записывается в память за шаг. **Noise_scale** (5% на ините, decay с differentiation) добавляет мультипликативный шум на i_gate при тренировке: `i_gate *= (1 + ε·noise_scale)`, ε ~ N(0,1). Это исследовательский шум — при diff=0 шум 5%, при diff=1 шум 0.1%. При τ≈150 общая память после 128 шагов:

```
mem[128] ≈ Σ(decay^t · i_gate · h[128-t]) ≈ i_gate · h · τ ≈ 0.047 · 7.0 ≈ 0.33·h
```

То есть mem_all имеет std ≈ 5-6, что здорово (не эксплодирует, не затухает). Для сравнения: если бы i_gate был exp(...) вместо sigmoid, mem_all бы имел std ≈ 640 на ините → NaN.

### 3.6 Grouped Cognitive Mirror

**Что делает.** Ансамбль из **G=32 независимых экспертов-зеркал**. Каждый эксперт живёт в своём d=112-мерном подпространстве (D=3584/32) и вычисляет **ограниченную, пер-дименсионную коррекцию** на основе **4 сигналов самосогласованности** в своём K=8-пространстве.

**Зачем группа, а не одно зеркало?** Одно Cognitive Mirror (K=16, D=3584) не могло дифференцироваться — var(log_scale) ≈ 0.000048 после 15000 шагов. Причина: tanh-насыщение убивает градиент к log_scale[d] = dL/d(mirror[d]) × mirror[d] — ноль в среднем по каждой размерности. 32 эксперта видят только свои 112 размерностей, каждый со своим K=8, tanh_bias и meta-gate — это даёт 32× больше градиентных сигналов независимо.

**Аналогия из жизни.** Представьте, что вы читаете текст. Ваш мозг постоянно делает микро-коррекции: "это слово не подходит по контексту", "эта конструкция грамматически странная", "я ожидал другое продолжение". 32 эксперта — это 32 разных "субличности": эксперт №7 специализируется на синтаксисе (видит размерности 784-895), эксперт №15 на семантике (размерности 1568-1679). Каждый эксперт может либо включиться (meta-gate ≈ 1), либо промолчать (meta-gate ≈ 0).

**Четыре сигнала коррекции:**

| Сигнал | Формула | Что измеряет | Куда в K-space |
|---|---|---|---|
| **temp** — темпоральный | `(hp - mc) * w_temp` | Отклонение от среднего VSA-памяти | lo (0…k/2-1) |
| **pred** — предиктивный | `(hp - W_pred @ hp_prev) * w_pred_scale` | Ошибка предсказания K-space (t-1 → t) | lo (0…k/2-1) |
| **smooth** — сглаживающий | `hp - conv1d(hp)` | Локальная негладкость K-space | hi (k/2…k-1) |
| **sym** — симметричный | `(hp·u)·(hp_prev·v)` | Асимметрия эволюции во времени | hi (k/2…k-1) |

**Frequency-Adaptive K (разделение lo/hi):**

Первые `k/2 = 4` направления K-space отведены под **медленные сигналы** (temp + pred) — долгоживущие ошибки самосогласованности. Последние `k/2 = 4` — под **быстрые сигналы** (smooth + sym) — локальные дрожания и асимметрии. Разделение не имеет дополнительных параметров — это структурное свойство K-пространства.

**Predictive Mirror (self-model):**

```python
# W_pred (G, k, k) — линейный предсказатель K-space
pred_k = einsum('blgk,gkk->blgk', hp_prev, W_pred)   # предсказание hp[t] из hp[t-1]
pred_error = (hp - pred_k) * w_pred_scale              # сигнал коррекции
# Кэш для auxiliary loss (собирается WideBindStack.forward)
self._cached_pred_k = pred_k
self._cached_hp = hp
```

Зеркало учится предсказывать собственное K-space. Чем лучше предсказание, тем меньше pred_error — модель «спокойна». Чем хуже — тем сильнее коррекция.

**W_pred weight_decay=0.** В `gate_no_decay` группе. Причина: per-param градиент W_pred усредняется через einsum по B×L, что даёт в 42× меньший per-param сигнал чем у pred_scale. Weight decay (0.01) пересиливал градиент (3:1). Без wd=0 W_pred не может выйти из identity.

**Auxiliary loss.** MSE(pred_k, hp.detach()) с weight=0.01 собирается со всех 32 слоёв. При ините даёт ~50% градиента к W_pred в обход gate. Без него W_pred получал бы градиент только через gate (заблокирован при насыщении |pred_error| путь через gate_signal → w_gate).

**Архитектура каждого эксперта:**
```python
# Каждый из 32 экспертов:
d = 112          # размер подпространства
k = 8            # размер K-space

# 1. Проекция в K-space
hp = h_g @ W_proj[g]            # (B, L, 112) → (B, L, 8)
mc = mem_centroid @ W_proj[g]   # среднее памяти в K-space
hp_prev = pad(hp[:, :-1])       # сдвиг на 1 для t-1

# 2. Четыре сигнала
temp = (hp - mc) * w_temp                   # lo
pred_error = (hp - W_pred @ hp_prev) * w_pred_scale  # lo
smooth = hp - conv1d_depthwise(hp)          # hi
sym = (hp * w_u) * (hp_prev * w_v)         # hi

# 3. Frequency-Adaptive merge
k_lo = k // 2
delta_lo = temp[..., :k_lo] + pred_error[..., :k_lo]
delta_hi = smooth[..., k_lo:] + sym[..., k_lo:]
delta = cat([delta_lo, delta_hi], dim=-1)   # (B, L, 8)

# 4. Нормировка + tanh_bias
delta = rms_norm(delta, k=8)
delta = delta + tanh_bias[g]                # ломает симметрию

# 5. Обратная проекция + skip connection
linear = delta @ W_out[g]                   # (B, L, 8) → (B, L, 112)
mirror_g = tanh(linear) + α_g * linear      # α_g = exp(log_skip_alpha[g]) — per-expert learned!
mirror_g = mirror_g * exp(log_scale[g])     # пер-дименсионная амплитуда
```

**K-Space Gate (per-token, per-expert):**

Изначально meta-gate использовал h_pool (G, d=112) — пул входного подпространства эксперта. Это давало `var(log_scale) ≈ 0.0025` с плато: gate был «слеп», потому что h_pool — это **вход** эксперта, а не его выход.

```python
# Было: gate от h_pool (d=112, pooled over B×L)
h_pool = mean(h_g, dim=(B, L))                    # (G, d)
gate_logit = h_pool @ w_gate + b_gate               # w_gate: (G, d)
expert_gate = sigmoid(gate_logit)                  # (G,) — один gate на эксперта
```

**Проблема:** градиент `∂L/∂w_gate[i] = h_pool[i] · g_i(1-g_i) · качество_эксперта_i`. h_pool имеет 112 шумных измерений. SNR на параметр: 1/112 ≈ 0.009. W_pred (учится быстрее всех) имеет 64 параметра на эксперт и 256 токенов градиента — SNR в 445× выше.

**Catch-22 и его исправление (сессия 2026-07):**

Тренировка с `gate_signal = hp + β·pred_error` выявила проблему: W_pred не выходил из identity через 10000 шагов. Анализ показал три причины:

1. **Gate насыщается** при trained weights (sigmoid_deriv=0.042 vs 0.198). hp (шум, не коррелирует с loss) доминирует над pred_error.
2. **Weight decay (0.01) в 3× сильнее градиента** W_pred — толкает параметры к нулю, а не к обучению.
3. **Gradient per-param в 42× меньше** чем у pred_scale (einsum усредняет по B×L).

**Исправления:**

```python
# Стало: gate от |pred_error| (k=8, per-token over B×L)
gate_signal = torch.abs(pred_error)                 # (B, L, G, k) — |pred_error| большой = плохое предсказание
gate_logit = einsum('blgk,gk->blg', gate_signal, w_gate) + b_gate
expert_gate = sigmoid(gate_logit)                   # (B, L, G) — per-token!
mirror = mirror * expert_gate.unsqueeze(-1)         # (B, L, G, d)
```

Изменения:
- **Gate: hp + β·pred_error → |pred_error|** — gate открывается когда предсказание плохое (|pred_error| большой), закрывается когда хорошее. Градиента к W_pred в **32× больше** vs старая формула (при trained W_proj).
- **W_pred weight_decay=0** — moved в `gate_no_decay` группу. Предотвращает подавление слабого градиента.
- **Auxiliary loss** — MSE(pred_k, hp.detach()) с weight=0.01, собирается со всех 32 слоёв. Обеспечивает ~50% градиента к W_pred в обход gate.
- **gate_pred_scale удалён** — параметр не использовался в новой формуле gate (|pred\_error|).

**Проверено на синтетических AR-1 данных:** W_pred |I-diff| вырос с 0.0084→0.0322 (4×), диагональ сошлась к истинному AR-коэффициенту (0.801, target=0.8).

**delta_var** — adaptive EMA: `α = delta_var_ema_min + diff·(delta_var_ema_max - delta_var_ema_min)`. Default: min=0.80, max=0.99. При low diff: α≈0.8 (быстрое обновление), при high diff: α≈0.99 (медленное). Эксперт с высокой variance активно корректирует — gate открыт. С variance ≈ 0 — либо выключен, либо коллапсировал — gate прижат. Параметры из config: `delta_var_ema_min`, `delta_var_ema_max`.

**Skip connection (per-expert learned α_g = exp(log_skip_alpha[g]), init 0.1):**
```python
skip_alpha = exp(log_skip_alpha)            # (G,) per-expert learned
mirror = tanh(linear) + skip_alpha[g] * linear
```
Без него: ∂L/∂log_scale[d] = ∂L/∂mirror[d] × mirror[d]. Когда tanh насыщен (linear ≫ 1), mirror[d] ≈ ±1, градиент ≈ 0 в среднем. log_scale не двигается → var(ls) = 0 → дифференциация зеркала не начинается.

С skip: mirror = tanh(linear) + α_g·linear. Производная: dmirror/dlinear = (1 - tanh²) + α_g ∈ [α_g, 1+α_g]. Всегда есть линейная компонента → градиент к log_scale не зануляется. **Каждый эксперт учит свой α_g** — одни эксперты могут сильнее полагаться на skip, другие меньше.

**Что происходит на практике (D=3584, 32 эксперта, k=8, K-space gate):**

| Шаг | var(log_scale) | i_gate | τ | |mirror| | W_pred.std | Статус |
|-----|---|---|---|---|---|---|---|
| 0 | 0.0025 | 0.50 | 224-348 | 0.0 | 0.01 | Инит (N(0,0.05), K-space gate) |
| 1000 | 0.0025 | 0.50 | 224-348 | — | — | Warmup конец |
| 2000 | 0.0025 | 0.50 | 224-348 | 0.21 | 0.35 | MirrorLR включён, W_pred растёт |
| 3000 | 0.0025 | 0.50 | 224-348 | 0.21 | — | K-space gate внедрён |

**До K-space gate (h_pool, d=112):** var(ls) платонуло на 0.0025 — gate «слепой», SNR 0.009/шаг.
**После K-space gate (hp, k=8):** SNR = 32/шаг (3555×). W_pred (std=0.35, растёт) — driver дифференциации. Ожидание: var(ls) растёт через 500-1000 шагов.

**tanh_bias — зачем bias в K-space?** Каждый эксперт инициализируется со своим bias в K=8 пространстве:
```python
self.tanh_bias = nn.Parameter(torch.zeros(G, k))  # (32, 8)
```
Даже при нулевом входе (delta=0), tanh_bias даёт разный выход каждому эксперту. Это разрывает симетрию: без него все 32 эксперта на ините идентичны — градиент не знает, какого эксперта менять.

**Per-expert learned parameters (5 новых параметров на эксперт):**

```python
# Каждый эксперт учит свои:
log_skip_alpha = nn.Parameter(tensor(0.0))        # α = exp(log_skip_alpha), init 0.1
log_dvar_mod_scale = nn.Parameter(tensor(-2.0))   # gate mod от delta_var, init ~0.14
dvar_mod_bias = nn.Parameter(tensor(-0.01))        # bias для delta_var mod
log_grad_mod_scale = nn.Parameter(tensor(-2.0))    # gate mod от grad_norm, init ~0.14
grad_mod_bias = nn.Parameter(tensor(-0.01))         # bias для grad_norm mod
```

Раньше gate modulation была хардкодом `0.1·tanh(x-0.01)` — одинаковым для всех экспертов. Теперь каждый эксперт учит свою чувствительность к variance и gradient-norm. Разные эксперты могут по-разному реагировать на одни и те же сигналы (например, эксперт №3 может быть чувствителен к variance, а эксперт №17 — к градиенту).

log_skip_alpha: в экспоненциальной параметризации, чтобы α>0 всегда. init=0.1 — слабый skip. По мере обучения некоторые эксперты могут увеличить α до 0.5+ (больше linear-компоненты), другие остаться на 0.1.

**Связь с AdaptiveController.** `|mirror|` усредняется по всем 32 экспертам перед передачей в контроллер. `var(log_scale)` — тоже по всем экспертам. Контроллер видит агрегированный сигнал и подстраивает гейты памяти глобально, а эксперты дифференцируются локально.

### 3.7 Grouped MLP

**Что делает.** Основной потребитель параметров (92.7%). Заменяет стандартный MLP (D→D→D) на групповую структуру: D=3584 делится на 32 группы по 112, каждая с 8× внутренним расширением.

```
# Вход: h (B, L, D=3584)

# RMS Norm (как в flat MLP)
h = rms_norm(h, (D=3584,), norm_w)

# Разбивка на группы
h = h.reshape(B, L, G=32, d=112)

# Пер-групповой MLP (независимый!)
for g in range(G):
    h_up = silu(h_g @ W_up[g])    # 112 → 896 (8× расширение)
    h_down = h_up @ W_down[g]     # 896 → 112

# Обратная разбивка
h = h.reshape(B, L, D=3584)
```

Реализуется как один einsum:

```python
h = F.silu(torch.einsum('blgd,gdf->blgf', h, W_up))   # (B, L, 32, 896)
h = torch.einsum('blgf,gfd->blgd', h, W_down)           # (B, L, 32, 112)
h = h.reshape(B, L, D)
```

**Сравнение конфигураций:**

| Конфигурация | Параметров | Промежуточных нейронов | Эффективный ранг (L23) |
|---|---|---|---|
| Flat D→D→D (3584→3584→3584) | 25,690,112 | 3584 | 8-12 (0.3%) |
| **Grouped G=32, expand=8** | **6,422,528** | **28,672** | **70-90 (63-80%)** |

Grouped MLP даёт **в 8× больше промежуточных нейронов** при том же числе параметров. В flat MLP число промежуточных нейронов = D (количество не растёт с расширением — это не bottleneck, а равномерный MLP). В Grouped MLP: 32 группы × 896 = 28,672 нейрона — в 8× больше.

**Эмпирика: почему группы работают лучше?**

На шаге 15000:

| Слой | Flat MLP (eff_rank/D) | Grouped G=32 (eff_rank/d) |
|---|---|---|
| L0 | 320/3584 (9%) | 96/112 (86%) |
| L5 | 180/3584 (5%) | 91/112 (81%) |
| L10 | 95/3584 (2.7%) | 85/112 (76%) |
| L15 | 42/3584 (1.2%) | 74/112 (66%) |
| L20 | 18/3584 (0.5%) | 52/112 (46%) |
| L23 | 8/3584 (0.2%) | 35/112 (31%) |

Grouped MLP держит эффективный ранг на уровне 31-86% от d=112 даже на глубоких слоях, где flat MLP коллапсирует до 0.2%.

**Почему группы не мешают друг другу?** Потому что каждая группа оперирует в своём 112-мерном подпространстве. Группа 17 может специализироваться на синтаксисе, группа 23 на семантике, группа 5 на пунктуации — они не конкурируют за параметры.

### 3.8 DCT Spectral

**Что делает.** Применяет обученное пер-дименсионное частотное масштабирование к h через дискретное косинусное преобразование (DCT-II).

```python
V = dct_basis(D=3584)                    # DCT-II матрица (3584×3584), ортогональная
h_dct = h @ V.T                          # (B, L, 3584) → DCT коэффициенты
h = h + (h_dct * lambda_k) @ V           # масштабирование + обратное DCT
```

**lambda_k** — обученный D-мерный вектор, инициализированный линейно:

```python
lambda_k[layer] = 0.5 + layer_idx / (n_layers - 1)
```

Для 32 слоёв: L0 = 0.5 (все частоты подавлены вдвое), L16 = 0.984 (почти нейтрально), L31 = 1.5 (все частоты усилены в 1.5×).

**Что lambda_k учит?** Каждый из 3584 DCT-компонентов имеет свой коэффициент масштабирования. Модель может:
- Усилить высокие частоты (детали, редкие паттерны)
- Подавить низкие частоты (шум, общий фон)
- Сделать частотно-избирательное усиление (одни частоты поднять, другие опустить)

После тренировки lambda_k на глубоких слоях (L26-L31) имеет per-dim std 0.01-0.05 — модель дифференцирует частоты, но не агрессивно. На ранних слоях (L0-L5) lambda_k практически константа — все частоты обрабатываются одинаково.

**Гипотеза специализации слоёв:**

| Слои | lambda_k | Режим | Предполагаемая роль |
|---|---|---|---|
| L0-L6 | 0.50-0.69 | Все частоты подавлены | Контекстная интеграция — построение стабильного представления |
| L7-L16 | 0.71-1.02 | Почти единица | Синтаксис — все частоты одинаково важны |
| L17-L24 | 1.03-1.27 | Умеренное усиление | Лексика — начинает подчёркивать паттерны |
| L25-L30 | 1.31-1.47 | Сильное усиление | Детали — дискриминативные признаки |
| L31 | 1.50 | Максимальное усиление | Сжатие для LM head |

### 3.9 LM Head (Partitioned Readout)

**Что делает.** Проецирует финальное h (D=3584) в логиты для 50000 токенов через **те же 32 сегмента**, что и эмбеддинг.

```python
h_final = rms_norm(h, (D,), final_norm_w)     # финальная RMS нормировка
# Разбиение на 32 сегмента × 112
for k in range(32):
    h_k = h[:, :, k*112:(k+1)*112]
    score_k = ⟨h_k, r_k⟩                        # скаляр — согласие сегмента и readout
logit_v = Σ_k z_{vk} · score_k                  # только активные для v биты
```

**Ключевое свойство:** голова не имеет cross-talk. Бит k читает ТОЛЬКО сегмент k финального h. Никакого `⟨h₃, r₇⟩` — каждый сегмент независимо голосует за свой бит. 

Это принципиально меняет динамику обучения:
- Информация должна быть **организована по сегментам** к последнему слою
- Bind (K=32), mirror (32 группы) и MLP (32 группы) — единственные каналы кросс-сегментного обмена
- Модель не может «схитрить» и перемешать всё в голове

**Сравнение:**

| Подход | Параметров | Cross-talk | Сегментная организация |
|---|---|---|---|
| Learned embedding | 3584×50000 = 179.2M | полный | нет |
| Linear Zeckendorf | 3584×23 + 23×50000 = 1.17M | полный (D→K) | нет |
| **Partitioned Head** | **32×112 = 3584** | **нет** | **принудительная** |

Параметров: 3584 против 1.17M — фактор 327× экономии. Но главное не экономия, а структурное принуждение: h на последнем слое обязан быть организован по 32 сегментам, иначе head не может прочитать информацию.

**Как градиент течёт:** `∂L/∂logit_v = error_v`. `∂L/∂r_k = Σ_{v: z_k=1} z_{vk} · error_v · h_k`. Каждый r_k получает градиент только от токенов с z_k=1 — ~9375 токенов из 50000. Остальные 40625 не шумят в r_k. Это **градиентная группировка** — каждый readout учится от своего непересекающегося подмножества словаря.

**Сегменты и residual:** через 24 слоя gradient от r_k (только z_k=1) распространяется обратно ко всем компонентам. Bind, mirror и MLP вынуждены научиться направлять информацию от любых сегментов к тем, которые активирует целевой токен — это самодостаточный механизм организации скрытого пространства.

### 3.10 Adaptive Controller

**Что делает.** Полностью автоматическая подстройка всех гиперпараметров VSA-памяти. Никаких ручных learning rate для гейтов, никакого grid search. Всё управляется двумя сигналами из Cognitive Mirror.

**Два фундаментальных сигнала:**

```python
exploration = min(1, |mirror| / 0.3)
    # Насколько сильно зеркало корректирует выход.
    # Высокое → модель активно подстраивается, нужна агрессивная память.
    # Низкое → модель стабильна, нужна консервативная память.

differentiation = min(1, var(log_scale) / 0.1)
    # Насколько зеркало специализировалось пер-дименсионно.
    # Высокое → зеркало знает, какие размерности подавлять/усилять.
    # Низкое → зеркало ещё исследует.
```

**Пять адаптируемых параметров:**

| Параметр | Диапазон | От чего зависит | Формула | Config поля |
|---|---|---|---|---|
| b_d (bias decay) | [2.0, 5.0] → τ ≈ [8, 150] | exploration + layer | `(2.0+3.0·layer_frac) + expl·(5.0 - ...)` | fill_, иерархия τ, cap=5.0 |
| b_i (bias write gate) | [-3.0, -1.5] → i_gate ≈ [0.047, 0.27] | exploration | `-3.0 + expl·1.5` | fill_, сужение, i_gate≤0.27 |
| w_mem2v_scale | [min, max] → [0.5, 1.0] | differentiation | `max - diff·(max-min)` | `w_mem2v_scale_min/max` |
| EMA α (global state) | [min, max] → [0.90, 0.99] | differentiation | `min + diff·(max-min)` | `ema_alpha_min/max` |
| EMA α (δ_var gate) | [min, max] → [0.80, 0.99] | differentiation | `min + diff·(max-min)` | `delta_var_ema_min/max` |
| Noise scale (гейты) | [min, max] → [0.001, 0.05] | differentiation | `max - diff·(max-min)` | `noise_scale_min/max` |

**Ключевое изменение:** b_d теперь **пер-слойный** — L0(τ≈8) → L31(τ≈150 на expl=0). Ранние слои быстро забывают (короткий контекст), глубокие — медленно (длинный контекст). Верхняя граница b_d=5.0 (τ≈150) — ограничивает перегрузку памяти при высоком exploration.

b_i сужен: `-3.0 + expl·1.5` (было `-3.0 + expl·3.0`). При expl=0: i_gate=0.047, при expl=1: i_gate=0.27. Старая формула давала i_gate=0.5 при expl=1 → перегрузка VSA-памяти (||mem||² ~ 26, целевой ~5-6). Новая — i_gate ≤ 0.27, память в норме.

**Gate LR:** gate_lr_mult=5.0. W_pred — в `gate_no_decay` (weight_decay=0, LR×5). Причина: per-param градиент W_pred в 42× меньше чем у pred_scale, weight decay (0.01) пересиливал градиент (3:1). WD=0 — essential fix.

**Интуиция:**

- **exploration высокий** (зеркало делает большие коррекции) → Память должна быть короткой (низкий τ/high b_d → decay быстрый) и писать агрессивно (high b_i → много информации). Почему? Потому что зеркало говорит "модель активно учится, старые данные в памяти устаревают быстро".

- **differentiation высокая** (зеркало специализировалось) → Память должна уступить зеркалу (низкий w_mem2v_scale → меньше памяти в выходе) и обновлять свою self-модель медленно (высокий EMA α). Почему? Потому что зеркало уже знает, какие размерности важны, и не нуждается в частых пересмотрах.

**Интеграция в forward:**

```python
# WideBindStack.forward():
expl, diff = AdaptiveController.stats(layers,      # thresholds из config
    expl_thresh=cfg.exploration_threshold,
    diff_thresh=cfg.differentiation_threshold)
n = len(layers)
for i, layer in enumerate(layers):
    layer_frac = i / max(n - 1, 1)
    b_i_val = -3.0 + expl * 1.5
    b_d_val = (2.0 + 3.0 * layer_frac) + expl * (5.0 - (2.0 + 3.0 * layer_frac))
    layer.b_i.fill_(b_i_val)                         # ! fill_ — не grad!
    layer.b_d.fill_(b_d_val)
mem2v_scale = AdaptiveController.w_mem2v_scale(layers,   # range из config
    min_val=cfg.w_mem2v_scale_min, max_val=cfg.w_mem2v_scale_max)
ema_alpha = AdaptiveController.ema_alpha(layers,         # range из config
    min_val=cfg.ema_alpha_min, max_val=cfg.ema_alpha_max)

# WideBindBlock.forward():
enhanced = bind_out + mem_read * w_mem2v * mem2v_scale + mirror
```

Обратите внимание: `.fill_()` устанавливает bias напрямую, минуя градиент. b_i и b_d исключены из optimiser parameter groups (`param_groups()` не включает их). Это предотвращает конфликт между адаптивным контролем и градиентными обновлениями.

W_pred — в `gate_no_decay` (weight_decay=0, LR×5). Per-param градиент W_pred (усредняется через einsum по B×L) в 42× меньше чем у pred_scale (поэлементный). С weight_decay=0.01 градиент не мог преодолеть wd. WD=0 — ключевое исправление catch-22. w_pred_scale в `gate_no_decay` (WD=0, LR×5) — наравне с W_pred, так как его градиент также ослаблен через цепочку |pred_error| → gate → mirror.

### 3.11 MirrorLRScheduler

**Что делает.** Модулирует learning rate на основе состояния Cognitive Mirror, с fallback на косинусное затухание.

**Два сигнала для LR:**

1. **var(log_scale)** — дисперсия пер-дименсионной амплитуды. Растёт от 0 (зеркало не дифференцировано) до ~0.1+ (специализировалось). LR падает пропорционально var / target_var.

2. **|mirror|** — средняя абсолютная коррекция. Большая → модель нестабильна, LR должен быть ограничен. Маленькая → модель сошлась, LR можно резать.

**Формула:**

```python
if var < var_min_for_lr_decay:               # config: 0.001
    mirror_mult = 1.0                        # зеркало не проснулось — не режем LR
else:
    decay = 1.0 - min(1.0, var / target_var) # config: 0.1
    mag_factor = min(1.0, max(lr_min_ratio, mag / mag_threshold))  # config: 0.05, 0.3
    mirror_mult = max(lr_min_ratio, decay * mag_factor)

# Fallback: forced cosine decay (floored at lr_min_ratio)
progress = step / max_decay_steps            # config: 50000
forced = max(lr_min_ratio, 0.5 * (1 + cos(π * min(1, progress))))
mult = min(mirror_mult, forced)              # берём минимум из двух
```

**Почему fallback?** Если зеркало заморожено (var < var_min_for_lr_decay всю тренировку), mirror_mult всегда 1.0, и LR не упадёт — модель никогда не сойдётся. Cosine fallback гарантирует, что LR всё равно уменьшится до lr_min_ratio от base_lr к концу тренировки.

**Конфигурация:** Все параметры — из WideBindConfig: `target_var`, `mag_threshold`, `lr_min_ratio`, `max_decay_steps`, `var_min_for_lr_decay`. Создаётся через `MirrorLRScheduler(model, optimizer, cfg=cfg)` — сам читает cfg.

## 4. Тренировка

### Данные

Токен-потоки хранятся в `.bin` файлах как uint16 массивы (numpy). Тренировка циклически перебирает все потоки:

```python
streams = [np.fromfile(f, dtype=np.uint16) for f in glob('token_stream_*_clean.bin')]
# Каждый поток: uint16 массив длиной 1.1B-1.8B токенов
```

Файлы `_clean.bin` — это очищенные версии (удалены перемежающиеся PAD токены из оригинальной записи).

**Форматы файлов:**

| Файл | Размер | Токенов | Источник |
|---|---|---|---|
| token_stream_ACTION_clean.bin | 1.1 GB | 549M | Действия (параллельные тексты) |
| token_stream_DETECT_clean.bin | 1.8 GB | 881M | Детектив (художественная литература) |
| **Total** | **2.9 GB** | **1.43B** | |

uint16 → uint16: 2 байта на токен → 2.9 GB RAM. При обучении на Colab (T4 16GB) это влезает в RAM с запасом.

### Батчи

```python
def get_batch(stream, seq_len, batch_size, offset):
    needed = B * L + 1
    if offset + needed > len(stream):
        offset = 0  # конец потока → начало
    chunk = stream[offset:offset + needed]
    x = chunk[:B*L].view(B, L)      # вход
    y = chunk[1:B*L+1].view(B, L)   # target (сдвиг на 1)
    return x, y, offset + B*L
```

**B=2, L=128** — максимальный батч для Colab T4 при D=3584 (221M). MX550 2GB — inference только. Colab T4 с auto-batch даёт B=2. 256 токена на шаг.

### Оптимизатор

```python
optimizer = torch.optim.AdamW(
    param_groups, lr=3e-4, betas=(0.9, 0.95), weight_decay=0.01
)
```

**Parameter groups:**

- **Decay (weight_decay=0.01):** Все 2D параметры (матрицы — W_proj, W_out, W_up, W_down, conv.weight и т.д.)
- **No decay (weight_decay=0):** Все 1D параметры (biasы, gate-вектора, scale-вектора, norm-вектора)
- **Excluded:** b_i, b_d (адаптивный контроллер устанавливает их напрямую)

### Scheduler

Две опции: MirrorLRScheduler (умный, адаптивный) или стандартный cosine (на случай если зеркала не проснулись).

### Eval

```python
for stream in streams:
    offset = len(stream) // 4       # вторая четверть потока (hold-out)
    for _ in range(n_batches):
        x, y, offset = get_batch(...)
        h = model.embed_tokens(x)
        out, state = model(h, state)
        loss = model.compute_loss(out, y)
```

**10 warm-up батчей** — пропускается без замера, чтобы state + AdaptiveController стабилизировались.

**100 батчей на каждый из N потоков** — измерение.

### Loss

```python
logits = self.lm_head(h)
ce_loss = F.cross_entropy(logits.reshape(-1, vocab), targets.reshape(-1))

# W_pred auxiliary loss: MSE(pred_k, hp.detach()) per layer
pred_loss = sum(F.mse_loss(pred_k[i], hp[i].detach()) for i in range(n_layers))
loss = ce_loss + pred_weight * pred_loss / n_layers   # pred_weight=0.01 default
```

Стандартная кросс-энтропия через partitioned head (см. 3.9). Дополнительно: **auxiliary loss** — MSE между предсказанием K-space (pred_k = W_pred @ hp_prev) и истинным состоянием (hp), усреднённый по всем 32 слоям. Weight=0.01 — мал, чтобы не доминировать над CE, но достаточен, чтобы обеспечить ~50% градиента к W_pred на старте (когда gate |pred_error| ещё слаб).

`compute_loss(h, targets, pred_weight=0.1)` — по умолчанию pred_weight=0.1. 2 аргумента = CE+aux, 3 аргумента = override.

### Генерация

```python
for _ in range(max_new_tokens):
    ctx = tokens[-L:].unsqueeze(0)     # контекст не длиннее seq_len
    h = model.embed_tokens(ctx)
    out, state = model(h, state)       # recurrent state переносится
    logits = model.lm_head(out[:, -1, :])  # только последний токен
    
    logits = logits / temperature
    if top_k > 0:
        vals, _ = torch.topk(logits, top_k)
        logits[logits < vals[:, -1:]] = -float('inf')
    
    probs = F.softmax(logits, dim=-1)

```

### FCF-CPR сжатие

Алгоритм крайне простой:

1. **Определение removable тензоров.** V_dct (D×D) и codes (vocab×K) — детерминированные, вычисляются заново из конфига.
2. **Удаление.** Скипаются полностью.
3. **Uniform 8-bit quantization.** Каждый оставшийся тензор независимо квантизуется в uint8: `index = round((value - min) / scale)`, где `scale = (max - min) / 255`. Хранится: uint8 тензор + 2 float (min, scale).
4. **Runtime-буферы** (`_last_gates`, `_last_h_pool`, `_prev_grad_norm`, `_last_magnitude`) — помечены `persistent=False`, не сохраняются в чекпоинт. Восстанавливаются на ините модели.

**Важно:** `decompress_sd` регенерирует `sparse_block_codes(vocab, K=cfg.code_dim, S=cfg.code_sparsity)` — не `zeckendorf_codes` из старой версии. При загрузке чекпоинта, сохранённого старой версией FCF-CPR, требуется патч (встроен в `colab.ipynb`, ячейка [4]).

**Ошибка:** MSE 1.7e-5 (наиболее пострадавший: W_proj, MSE 3.2e-5). Для val_loss: разница < 0.001.

## 9. Математическое приложение

### A. Доказательство ассоциативности prefix scan

Рекуррентность `mem[t] = a[t] · mem[t-1] + b[t]` можно записать как:

```
(mem[t], 1) = (a[t], b[t]) ⊗ (mem[t-1], 1)
            = (a[t] · mem[t-1] + b[t], 1)
```

Оператор ⊗ на парах (a, b) ассоциативен:

```
((a₁, b₁) ⊗ (a₂, b₂)) ⊗ (a₃, b₃)
= (a₁ · a₂, a₂ · b₁ + b₂) ⊗ (a₃, b₃)
= (a₁ · a₂ · a₃, a₃ · (a₂ · b₁ + b₂) + b₃)
= (a₁ · a₂ · a₃, a₂ · a₃ · b₁ + a₃ · b₂ + b₃)

(a₁, b₁) ⊗ ((a₂, b₂) ⊗ (a₃, b₃))
= (a₁, b₁) ⊗ (a₂ · a₃, a₃ · b₂ + b₃)
= (a₁ · a₂ · a₃, a₂ · a₃ · b₁ + a₃ · b₂ + b₃)

Same result. Ассоциативность выполняется → параллельный tree scan возможен.
```

### B. Почему bind работает (K=32) — градиентный анализ

```
forward: bind = (W_projᵀ h ⊙ w_u ⊙ w_v)ᵀ W_out
              = hᵀ W_proj · diag(w_u) · diag(w_v) · W_out
              = hᵀ M

где M = W_proj · diag(w_u) · diag(w_v) · W_out ∈ ℝᴰˣᴰ
```

Ранг M ≤ K (промежуточное представление K-мерно). При обратном распространении:

```
∂bind/∂h = Mᵀ  (полная D×D матрица, ранг K)
∂bind/∂W_proj = h ⊗ (diag(w_u) · diag(w_v) · W_out)  (внешнее произведение)
```

Важно: каждый элемент h влияет на все D элементов bind через K-мерный баттлнек. Якобиан M — не диагональный, в отличие от pure VSA.

### C. Effective Rank MLP

Effective rank (или stable rank) матрицы A ∈ ℝᵐˣⁿ с сингулярными числами σ₁ ≥ ... ≥ σ_min(m,n):

```
eff_rank(A) = (Σ σᵢ)² / Σ σᵢ²
```

Интерпретация: количество размерностей, несущих значимый сигнал.
- eff_rank = d = 112 → все размерности используются полностью
- eff_rank = 4 → только 4 из 112 размерностей несут сигнал, остальное шум

**Почему GroupedMLP имеет более высокий eff_rank чем Flat MLP?**

Flat MLP: `h_mlp = W_down · SiLU(W_up · h)`. rank(h_mlp) ≤ D = 3584. Внутреннее представление — 3584 нейрона (размерность входа в SiLU).

GroupedMLP: 32 группы, каждая с d=112 → 896× внутреннее представление. Общее число SiLU-нейронов: 32 × 896 = 28,672 — в 8× больше.

SiUL применяется 8× чаще, каждая на своём подпространстве. Это даёт 8× более богатые нелинейные взаимодействия без роста параметров.

### D. DCT-II ортогональность

Базис DCT-II V удовлетворяет:

```
V[n,k] = α_n · sqrt(2/D) · cos(π · n · (k + 0.5) / D)
```
где α₀ = 1/√2 и α_n = 1 для n > 0.

Ортогональность: `V @ Vᵀ = I`. Доказательство:

```
Σ_k V[n₁,k] · V[n₂,k]
  = (2/D) · Σ_k cos(π·n₁·(k+0.5)/D) · cos(π·n₂·(k+0.5)/D) · α_n₁ · α_n₂
  = δ[n₁, n₂]
```

Для n₁ = n₂ = 0: `Σ_k V[0,k]² = (2/D) · D · (1/2) = 1`
Для n₁ = n₂ > 0: `Σ_k V[n,k]² = (2/D) · Σ_k cos²(...) = 1`
Для n₁ ≠ n₂: интеграл косинусного произведения равен нулю по DCT-решётке.

### F. Масштабирование: геометрическая согласованность

WideBind спроектирован для масштабирования без потери структурной целостности. **Инварианты, которые нельзя нарушать:**

```
d = 112          — фиксирован при любом G (оптимально для 8× MLP, d%k==0)
D = G · d        — размерность выводится из числа групп
K = G = L        — bind-каналы = группы = слои (всегда)
k = 8            — фиксирован (d/k = 112/8 = 14 — целое)
S = max(4, round(K · 0.1875))  — ~18.75% активных бит
```

**Скелет масштабирования (d=112, k=8, expand=8, S=6):**

| G (=K=L) | D | Bind | MLP (92%) | Total | Tok/param (8B) | Hardware |
|---|---|---|---|---|---|---|
| 8 | 896 | 14K | 1.6M | **14M** | 571 | CPU/любой |
| 16 | 1792 | 57K | 3.2M | **55M** | 145 | MX550 |
| **32** | **3584** | **229K** | **6.4M** | **221M** | **36** | **MX550+T4** |
| 48 | 5376 | 516K | 9.6M | **503M** | 16 | T4/RTX 3090 |
| 64 | 7168 | 918K | 12.8M | **914M** | 8.7 | RTX 3090 |
| 96 | 10752 | 2.1M | 19.3M | **2.1B** | 3.8 | A100 |
| 128 | 14336 | 3.7M | 25.7M | **3.9B** | 2.1 | 2×A100 |
| 256 | 28672 | 14.7M | 51.4M | **17.5B** | 0.5 | H100 cluster |

**Что происходит при каждом удвоении G:**

| Компонент | Масштаб | Природа |
|---|---|---|
| D | ×2 | D = G·d |
| Bind | ×4 | O(D·K) = O(G²·d²) — квадратично |
| MLP | ×2 | O(G·d²·expand) — линейно по G |
| Mirror | ×2 | O(G·d·k) — линейно |
| Conv+VSA+Spectral | ×2 | O(D) — линейно |
| **Total** | **~×4** | **доминирует Bind на больших G** |

**Ключевой момент:** при G>64 Bind начинает догонять MLP:
```
G=32:  bind/MLP = 3.6%   (MLP рулит)
G=64:  bind/MLP = 7.2%
G=128: bind/MLP = 14.3%
G=256: bind/MLP = 28.6%  (Bind значим)
```

**Sequence length:** не нарушает геометрию. VSA — O(L log L) пер-слойно. Принципиальное преимущество WideBind: L не влияет на веса, только на активации. `seq_len` можно менять в любой момент без перетренировки:

| seq_len | VRAM (inf, fp16) | tok/s (MX550) | Замечание |
|---|---|---|---|
| 128 | 0.55 GB | 11 | Текущий |
| 512 | 0.65 GB | 8 | VSA scan ×4.5 |
| 2048 | 0.95 GB | 4 | Граница MX550 |
| 8192 | 1.80 GB | 1 | MX550 предел |

**code_sparsity (S):** S = 6 при K=32 даёт 6/32 = 18.75% активных бит. При масштабировании K, S должен расти пропорционально:
```
S(K) = max(4, round(K · 0.18))  — держать ~18% активность
K=32  → S=6
K=64  → S=12
K=128 → S=23
K=256 → S=46
```

**Процедура масштабирования (дорожная карта):**

```python
# 1. Выбрать G → вычисляем все производные размеры
G  = 64                               # цель
d  = 112                              # фиксирован
K  = G                                # bind каналы
D  = G * d                            # 7168
k  = 8                                # фиксирован
S  = round(K * 0.18)                  # ≈ 12
L  = G                                # слои = группы

# 2. Создать WideBindConfig — все числа самосогласованы
cfg = WideBindConfig(D=D, n_layers=G, bind_K=K,
                     mlp_groups=G, code_dim=K,
                     code_sparsity=S, mirror_k=k)

# 3. Инициализировать модель
model = WideBindStack(cfg)  # все выровнено 1:1

# 4. Загрузить чекпоинт меньшей модели (zero-shot transfer)
#    W_proj: (D_old, K_old) → (D_new, K_new) — interpolate
#    W_up:   (G_old, d, 8d) → (G_new, d, 8d) — pad new groups as zeros
```

**Перенос весов с G_old на G_new:**

| Компонент | Стратегия переноса |
|---|---|
| Bind (W_proj, W_out) | Интерполяция: D_old×K_old → D_new×K_new |
| MLP (W_up, W_down) | Новые группы = zero-init (не интерполировать) |
| Mirror (W_proj, W_out, W_pred) | Новые эксперты = копия ближайшего |
| w_gate | Zero-init для новых экспертов |
| VSA gates (w_d, w_i, ...) | Интерполяция D_old → D_new |
| PartitionedHead readout | Новые сегменты = zero-init |

**Итог:** масштабирование WideBind — это выбор одного числа G (число групп), из которого все остальные размеры выводятся автоматически. Никаких гаданий, никаких «попробуем 24 слоя, потом 32». Геометрия зафиксирована: `D = G·d = G·112`, `K = G`, `L = G`.

## 10. FAQ

**Почему модель генерирует только PAD?** Модель на 22000 шагов — это ~0.037 токена на параметр. Типичные LM требуют >1 токена на параметр для осмысленной генерации. После 300K шагов (~0.5 tok/param) генерация должна стать связной.

**Почему n_layers=32, а не 24?** D/L = 3584/32 = 112 = d (целое). L=24 давал D/L = 149.33 — архитектурный «остаток». С L=32 четыре partitionings совпали: L = G_mlp = G_mirror = K = 32. Каждый слой маппится 1:1 на группу/эксперта/код. τ_step = 10.2% (против 13.9% с L=24). Подробнее: раздел 9.

**Почему bind_K=32, а не 16?** bind_K=16 давал D/K = 224 (2 доли на канал). С K=32: D/K = 112 = d — каждый канал bind покрывает ровно одну MLP-группу. Alignment: K = G = L = 32. Eff_rank: 22-28/32 (69-88%).

**Почему D=3584, а не 4096 или 8192?** MX550 2GB VRAM. 221M в fp16 = 442 MB веса + 884 MB AdamW + градиенты ≈ 1.77 GB. + активации (~200-300 MB с L=32) = ~2.0 GB. На грани, но влезает. D=4096 даст ~252M — OOM.

**Почему не добавить MoE или sparse mixture?** MoE требует routing + auxiliary loss. GroupedMLP — "бедного человека MoE": 32 группы всегда активны, но каждая специализируется на своём подпространстве.

**Можно ли заменить sparse block codes на learned embedding?** Технически — да. Практически — 179M лишних параметров. Если когда-нибудь будет GPU с 24GB+, можно попробовать.

**Что будет после тренировки 300K шагов?** С K-space gate + bind_K=32 + L=32 ожидается:
- Train loss: ~3.0-4.0 (ppl 20-55) — 221M требует >1.5B токенов
- Осмысленная генерация коротких фраз после ~100K шагов
- var(log_scale): от 0.0025 → 0.01-0.05 (K-space gate разблокирует рост)
- W_pred: std → 1.0+ (предсказатель сойдётся)
- i_gate: 0.27 → 0.1-0.3 (адаптивный контроллер, i_gate ≤ 0.27)
- mirror_mult: 0.67 → 0.3-0.5 (LR затухает по мере роста var(ls))

**Как масштабировать архитектуру?** Весь скелет масштабирования — в разделе [9.F](#f-масштабирование-геометрическая-согласованность). Выбираешь G (= K = L), все размеры выводятся автоматически. При G=64 → 914M, при G=128 → 3.9B. Геометрия `D = G·112`, `G = K = L` не меняется никогда.
