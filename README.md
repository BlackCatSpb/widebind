# WideBind

**Языковая модель без единого transformer-слоя.** Никакого softmax, никакого attention, никаких QKV-проекций. **152 миллиона параметров, 32 слоя, размерность D=4096.**

Вместо attention — **VSA-память** (векторная суперпозиция с ассоциативным chunked prefix scan). Вместо multi-head projection — **bottleneck bind** (биллинейное скрещивание через K=64). Вместо Feed-Forward Network — **GroupedMLP** (32 параллельных MLP, каждый со своим 4× расширением). Вместо LayerNorm — **GroupedCognitiveMirror** (ансамбль из 32 экспертов-зеркал, каждый со своим K-пространством, 4 EMA-нормированными сигналами коррекции с learnable softmax-весами, **scalar alpha** per expert, **K-space gate**, φ-координатная init, geometrically init'ированные skip_α и mod_scales).

**Архитектура:** D=4096, 32 слоя, 32 эксперта зеркала, 32 группы MLP, каждая размерности d=128. Bind — скрещивание через bottleneck K=64. Эмбеддинг и голова — 32 сегмента × 128 с разреженным 6-out-of-32 кодом. `D = G·d = 32·128`.

**Ключевые механизмы:**
- **Chunked VSA Scan (CHUNK=32)** — параллельный prefix scan для численной стабильности на длинных последовательностях.
- **α-diag** — per-expert, per-k-dim скаляр вместо W_pred, градиент ×1024.
- **Bidirectional MirrorLR** — LR адаптируется к динамике специализации экспертов, без forced cosine decay.
- **Alpha Warmup Override** — форсированная дифференциация экспертов на warmup.
- **Surprisal-gated i_gate** — запись в VSA-память модулируется ошибкой предсказания.
- **Signal EMA Normalization** — 4 сигнала зеркала (temp/pred/smooth/sym) нормируются per-экспертными EMA и взвешиваются learnable softmax.
- **φ-coordinate init** — геометрическая инициализация per-expert скаляров по глубине.
- **VSA Long-Range Memory** — b_d per-channel до τ≈163K.
- **Per-layer Centroid EMA** — согласованная инерция центроида по глубине.
- **λ-d иерархия (d=3)** — 24 гиперпараметра выводятся из λ₃≈1.839.
- **Branch Balance Loss** — выравнивание дисперсий ветвей conv/bind/mirror.
- **tie_bind / tie_mirror_proj** — автоэнкодерное ограничение, по умолчанию включено.
- **PH+B (PartitionedHead + per-token bias)** — гибридный readout.

## 1. Философия: почему не Transformer

### Три структурных ограничения Transformer

**1. O(L²) attention.** Каждый токен смотрит на каждый другой токен. Для L=128 это 16,384 сравнений на голову — терпимо. Для L=8192 это 67 миллионов. Квадратичная стоимость вшита в архитектуру: чтобы получить attention, нужно вычислить softmax(QK^T / sqrt(d)) × V — произведение матриц размера (L×d) × (d×L) даёт O(L²) операций. Никакой аппроксимации это не убирает, потому что softmax требует нормировки по всем L токенам.

**2. Неограниченный KV-cache.** Инференс требует кэшировать все key-value пары для всех предыдущих токенов. Для 24 слоёв при D=896 и L=128K: 24 × 128K × 896 × 2 × 4 байта ≈ 22 GB — больше, чем у большинства GPU. Это означает, что генерация длинных последовательностей упирается не в compute, а в память. С каждым новым токеном cache растёт линейно, и после ~10K токентов даже A100 80GB начинает задыхаться.

**3. Softmax bottleneck.** Attention-веса нормируются через softmax, который производит распределение вероятностей. Это означает, что веса внимания обязательно положительные и в сумме дают 1. Теоретически известно, что это ограничивает представительную способность (известная проблема "softmax bottleneck" в seq2seq моделях). Модель не может выразить "этот токен важен с весом -1" — все веса лежат в (0, 1).

### Как WideBind решает каждое ограничение

| Проблема Transformer | Решение WideBind |
|---|---|
| O(L²) attention | **O(L log L) prefix scan** — ассоциативный параллельный скан над VSA-памятью |
| O(L) KV-cache на токен | **O(D) состояния на слой** — 16 KB для D=4096 (fp32), независимо от длины последовательности |
| Softmax bottleneck | **Биллинейный bind** через K=64 — скрещивание размерностей без вероятностных нормировок |

Три ключевых отличия:

**Память — это вектор, а не матрица.** В VSA-памяти каждый слой хранит один D-мерный вектор `mem[t]` (суперпозиция), а не матрицу K×V всех прошлых токенов. Новый токен "вписывается" в суперпозицию через взвешенное (гейтами) сложение, а старый постепенно затухает (decay). Это не attention — это **векторная рекуррентность**, которая параллелизуется через prefix scan.

**Скрещивание размерностей — через биллинейность, не через attention.** В attention каждая размерность "общается" с каждой через weighted sum токенов. В WideBind каждая размерность "общается" с каждой через проекцию в K=64, биллинейное произведение (u·v), и проекцию обратно. Это **не weighted sum**, а **покомпонентное произведение** — принципиально другая операция.

**Никакого softmax.** Во всей модели нет ни одной softmax-нормировки. Сигмоиды используются только для VSA-гейтов (decay и i_gate), где диапазон [0, 1] требуется по физическому смыслу рекуррентности. Tanh используется в Cognitive Mirror для ограничения коррекции [-1, 1]. Всё остальное — линейные проекции + SiLU.

## 2. Архитектура: общий поток

```
token IDs: (B, L) — целые числа от 0 до 49999
    │
    ├── PartitionedEmbedding (Sparse Block Codes, K=32, S=6)
    │     token → sparse code (ровно 6 из 32 бит) → сегменты 32×128 → h: (B, L, D=4096)
    │
    └── [WideBindBlock × 32]  (все с residual connection, L = G = 32)
          │
          ├── Pre-LN (RMS Norm)
          ├── Depthwise Conv1d (k=48, groups=D)
          ├── Bottleneck Bind (D → K=64 → diag(u)·diag(v) → D)
          ├── VSA Memory (chunked scan CHUNK=32, surprisal-gated i_gate, read + first moment)
          ├── h += enhanced (bind_out + mem_modulated + mirror)
          ├── Grouped Cognitive Mirror (32 экспертов × d=128, k_l∈{4,8,16}, φ-координата, 4 EMA-нормированных сигнала c softmax-весами, scalar alpha, K-space gate)
          ├── DCT Spectral (h_dct × lambda_k в базисе DCT-II)
          ├── Grouped MLP (32 группы × 128→512→128, SiLU, expand=4)
          └── h_out: (B, L, D)
    │
    ├── Final RMS Norm
    │
    └── PartitionedHead (32 сегмента × 128 readout)
          h → split(32×128) → ⟨h_k, r_k⟩ → Σ z_{vk}·score_k → logits: (B, L, 50000)
```

Визуализация одного блока с размерами (D=4096, G=32, bind_K=64, k∈{4,8,16}):

```
h: (B, L, 4096)
    │
    ├─── RMSNorm(4096)
    │
    ├─── Conv1d(groups=4096, k=48) ────┬─── h + conv
    │                                   │
    ├─── Linear(4096→64) → [bind] → Linear(64→4096) ─┐
    │   u = hp × w_u                                   │
    │   v = hp × w_v                                   │
    │   bind = (u × v) @ W_out                         │
    │                                                  │
    ├─── VSA Memory ───────────────────────────────────┤
    │   decay = sigmoid(h×w_d + b_d)   τ ≈ 8-400      │
    │   i_gate = softplus(h×w_i + b_i + γ·‖ê‖₂)   ~0.08│
    │   mem = chunked_prefix_scan(decay, h × i_gate)   │
    │         (CHUNK=32, стабилен при L≥128)            │
    │   read = mem × w_q + mu_read × w_mu_mem          │
    │                                                  │
    ├─── Grouped Cognitive Mirror ─────────────────────┤
    │   h → split(32 × 128)                            │
    │   for each expert g (φ-coordinate init):         │
    │     hp = h_g @ W_proj  (128→k_g)                 │
    │     pred_k = alpha_g * hp_prev                    │
    │     temp = (hp - mem_c) * w_temp                 │
    │     smooth = hp - causal_conv3(hp)               │
    │     sym = (hp×u) × (hp_prev×v)                  │
    │     signals = [temp, pred_error, smooth, sym]    │
    │     signals ← EMA-norm(signals)  per-expert RMS  │
    │     δ = Σ w_i · signals_i   softmax weights (4)  │
    │     mirror_g = [tanh + skip_α·linear](δ @ W_out) │
    │     mirror_g *= exp(log_scale) × gate_g          │
    │   mirror = merge(32 × 128 → 4096)                │
    │                                                  │
    │   h = h + bind + mem_read + mirror               │
    │                                                  │
    ├─── DCT Spectral ─────────────────────────────────│
    │   h_dct = h @ V_dct^T                            │
    │   h = h + (h_dct × lambda_k) @ V_dct            │
    │                                                  │
    └─── GroupedMLP ───────────────────────────────────┘
        h → reshape(G=32, d=128)
            for each group: SiLU(x@W_up)@W_down  (128→512→128)
        h → reshape(4096) + residual

h_out: (B, L, 4096)
```

**Зачем нужен каждый компонент?**

| Компонент | Роль | Что было бы без него |
|---|---|---|---|
| Conv1d | Локальный контекст (n-граммы) | Модель не видела бы локальных паттернов — bind+memory не захватывают соседние токены |
| Bottleneck Bind | Скрещивание размерностей | VSA-память работает по каждой размерности независимо → градиенты затухают после 4 слоёв |
| VSA Memory | Глобальный контекст (суперпозиция) | Модель видела бы только текущий токен + локальное окружение |
| Cognitive Mirror | Самокоррекция (32 эксперта, φ-init, 4 EMA-нормированных сигнала, learnable softmax-веса, K-space gate) | Модель не могла бы исправлять собственные ошибки — нестабильный выход |
| DCT Spectral | Частотная фильтрация | Все частоты обрабатывались бы одинаково — нет иерархии слоёв |
| GroupedMLP | Основная вычислительная мощность | ~93% параметров исчезли бы — модель не смогла бы учить сложные паттерны |

## 3. Компоненты: глубокое погружение

### 3.1 Partitioned Embedding (Sparse Block Codes)

**Что делает.** Преобразует token ID (0-49999) в D-мерные вектора через **сегментную адресацию**, а не через проекцию. D=4096 делится на K=32 сегмента по 128 размерностей — ровно как группы mirror и MLP.

**Как работает.** Каждый токен имеет детерминированный sparse код длиной K=32 бита, из которых ровно S=6 единиц. Код строится через **combinadic** (комбинаторную систему счисления): C(32,6)=906192 ≥ 50000, каждому токену назначается уникальная 6-элементная комбинация из 32. Единицы указывают, какие сегменты из 32×128 активировать:

```python
def sparse_block_codes(vocab=50000, K=32, S=6):
    total = comb(K, S)
    perm = torch.randperm(total, generator=manual_seed(42))
    codes = zeros(vocab, K)
    for v in range(vocab):
        n = int(perm[v])            # комбинаторный индекс
        for i in range(S, 0, -1):
            c = i - 1
            while comb(c + 1, i) <= n:
                c += 1
            codes[v, c] = 1.0
            n -= comb(c, i)
    return codes
```

Токен 42 → combinadic код с ровно 6 единицами из 32.
Токен 49999 → другой уникальный 6-битный код.

Спарс-код (K=32) адресует 32 сегмента по d=128:

```python
h[v] = sum(z_{vk} * basis_k)   # z_{vk} ∈ {0,1}: ровно 6 из 32
```

**Что было до sparse block codes (Zeckendorf, K≈23):**

```python
# 32 сегмента по 128 dim, каждый бит — свой basis vector
h[v] = Σ_k z_{vk} · w_k    (w_k ∈ ℝ¹²⁸, конкатенация)
# z_{vk} ∈ {0, 1}: ровно 6 из 32 бит активны
```

Токен «включает» 6 из 32 вентилей. Остальные 26 молчат. Это не размазывание по всем 4096 размерностям, а **адресация**: «мой код — эти 6 сегментов».

**Ключевое отличие от Linear-проекции:**

| Свойство | Linear K→D (было) | Partitioned (стало) |
|---|---|---|
| Ранг матрицы E | ≤ 23 | **4096 (полный)** |
| Per-token активных dims | 23 (0.6% от D) | **672 (18.8%, детерминированно)** |
| Частота бит | 7%–38% (5× разброс) | **18.2%–19.1% (равномерно)** |
| Gradient grouping | все 50000 токенов → один W_proj | **только токены с z_k=1 → w_k** |
| Параметров | 82,432 | 4,096 |
| Выравнивание с mirror | нет | **1:1 (32×128)** |

**Почему K=32?** 32 — естественное число архитектуры: mirror (32 группы), MLP (32 группы). Все компоненты используют d=128 (D/G). Для sparse кода C(32,6)=906192 ≥ 50000 — достаточно.

**Градиентная группировка.** Каждый w_k учится только от токенов с z_k=1. Это ~9375 токенов из 50000 — в 5× меньше шума, чем общая проекция. ∂L/∂w_k = 0 для всех токенов, где бит k неактивен — доказуемо ровный ноль, не «почти ноль».

**LM Head использует ту же структуру** — 32 сегмента × 128, каждый со своим readout r_k:

```python
logit_v = Σ_k z_{vk} · ⟨h_k, r_k⟩
```

Голова не имеет cross-talk: бит k читает ТОЛЬКО сегмент k финального h. Если информация для предсказания токена лежит не в его сегментах — 24 слоя WideBind обязаны её туда переорганизовать. Это **принудительная организация скрытого пространства**.

### 3.2 Pre-LN RMS Norm

**Что делает.** Нормирует вход каждого блока перед саблеерами. Использует RMS Norm — без вычитания среднего, только деление на среднеквадратичное.

```
RMS(x) = sqrt(mean(x²) + ε)
h = x / RMS(x) * w
```

где `w` — обученный вектор масштаба размерности D, инициализированный единицами.

**Почему RMS Norm, а не LayerNorm?**

LayerNorm сначала вычитает среднее, затем делит на стандартное отклонение. RMS Norm только делит на RMS. Для нуль-средних активаций это одно и то же, но без лишнего вычисления среднего (D операций на токен на слой). При 32 слоях и D=4096: 32 × 4096 = 131,072 лишних операций на токен. Немного, но без них тоже работает (Zhang & Sennrich, 2019).

**Почему pre-LN (перед саблеерами), а не post-LN (после)?**

В оригинальном Transformer (Vaswani et al., 2017) LayerNorm стояла после каждого саблеера (post-LN). Это вызывало нестабильность тренировки — residual путь проходил через нормировку, что амплифицировало градиенты. Pre-LN (Xiong et al., 2020) ставит норму перед каждым саблеером, оставляя residual путь ненормированным. Это текущий стандарт для всех современных архитектур.

### 3.3 Depthwise Conv1d

**Что делает.** Глубинная (depthwise) одномерная свёртка с kernel=48, независимо для каждого из D=4096 каналов.

```
h_perm = h.transpose(1, 2)     # (B, L, D) → (B, D, L)
h_conv = conv1d(h_perm)        # Conv1d(in=D, out=D, kernel=48, groups=D)
h_conv = h_conv.transpose(1, 2) # (B, D, L) → (B, L, D)
h = h + h_conv
```

**Параметры:** D × kernel = 4096 × 48 = 196,608 на слой. Для сравнения, полная свёртка (не depthwise) потребовала бы D² × kernel = 4096² × 48 ≈ 805M — в 4096× больше.

**Почему kernel=48?** 48 токенов из 128 покрывает ~37% последовательности. Для локального контекста этого достаточно: n-граммы до 48 токенов захватывают фразы, клаузы и короткие предложения.

**Почему depthwise, а не обычная?** Каждый из D=4096 каналов обрабатывается своим независимым фильтром. Разные каналы представляют разные семантические признаки (синтаксис, семантика, кореференция и т.д.), и у каждого признака своя временная динамика. Depthwise позволяет каждому каналу учить свой фильтр без интерференции от других каналов.

**Stateful Conv.** Свёртка поддерживает буфер состояния из предыдущих `kernel-1` шагов:

```python
if state is None:
    state = zeros(B, D, padding)  # 47 нулей на канал
h_perm = h.transpose(1, 2)
h_conv = conv(cat([state, h_perm], dim=-1))
h_conv = h_conv[..., :L].transpose(1, 2)
state_out = h_perm[:, :, -(padding):]
```

Это позволяет корректно обрабатывать последовательность через границы батчей — состояние переносится между вызовами `.forward()`.

### 3.4 Bottleneck Bind (D→K→D)

**Что делает.** Проецирует D=4096 в K=64, применяет биллинейное взаимодействие в K-space, проецирует обратно в D. Это **единственный механизм скрещивания размерностей** в WideBind.

```
hp = h @ W_proj          # (B, L, D=4096) → (B, L, K=64)
u = hp * w_u             # элемент-умножение на вектор w_u
v = hp * w_v             # элемент-умножение на вектор w_v
bind_out = (u * v) @ W_out  # (B, L, K=64) → (B, L, D=4096)
```

**Математика.** Полное преобразование: `bind_out = h @ W_proj · diag(w_u) · diag(w_v) · W_out`. Если обозначить M = W_proj · diag(w_u) · diag(w_v) · W_out, то `bind_out = h @ M`. Размер M — D×D (4096×4096), **ранг не выше K=64**.

**Почему это важно?** В pure VSA (векторная суперпозиция) каждый канал эволюционирует независимо:

```
mem[t] = decay[t] · mem[t-1] + h[t]    # (B, L, D) поэлементно
```

Якобиан ∂mem[t]/∂h[t] = I (единичная матрица). Градиент для каждого канала зависит только от этого канала — **никакого скрещивания**. Через 4 слоя градиенты затухают до нуля, потому что нет механизма "перекинуть" сигнал из канала i в канал j.

Bind решает это: **M — не диагональная матрица**. Её ранг всего 64, но все D×D = 16.8M элементов матрицы M соединены через 64-мерный баттлнек. Градиент от bind_out[i] течёт ко всем h[j] через M[i,j].

**Эмпирика скрещивания (D=4096, K=64):**

| Компонент | Grad/param на ините | Эффект |
|---|---|---|
| Pure VSA (без bind) | < 1e-6 | Затухание после 4 слоёв |
| Bind K=1 | ≈ 0.01 | Слабое скрещивание |
| **Bind K=64** | **≈ 0.43** | **Здоровый градиент через все 32 слоя** |
| Bind K=4096 (полный) | ≈ 0.9 | Максимальное скрещивание, но ×6000 парам. |

**Почему K=64, а не 32?**

| K | Параметры (bind) | Grad/param | Эффективный ранг |
|---|---|---|---|
| 4 | 32,800 | 0.15 | 3.8 |
| 8 | 65,600 | 0.28 | 7.2 |
| 16 | 131,136 | 0.42 | 12-14 |
| 32 | 262,208 | 0.51 | 22-28 |
| **64** | **524,352** | **0.57** | **38-50** |
| 128 | 1,048,640 | 0.63 | 45-60 |
| 4096 | 33,558,528 | 0.89 | 896-1792 |

K=64 выбран из соображений ёмкости: D/K = 64, эффективный ранг 38-50/64 (59-78%). При tied проекциях экономия `64×4096 = 262K` параметров.

**Инициализация — критический момент.** w_u и w_v инициализируются с std=1.0 (не 0.02, как дефолтный PyTorch Linear). Причина: произведение w_u × w_v × W_out масштабируется как std³. При std=0.02, градиент на ините = 0.02³ = 8e-6 — эффективный ноль. При std=1.0, градиент = 1.0³ = 1.0 — полный градиент. Разница в 125,000×.

**tie_bind (по умолчанию True):** При включении `W_out = W_proj^T` — автоэнкодерное ограничение. `W_out` становится буфером (не параметром), синхронизированным с `W_proj.T` через forward_pre_hook. Градиент течёт только в `W_proj`. Экономия `bind_K × D = 64×4096 = 262K` параметров. Бенчмарк: почти без потери loss (−0.22 на D=256), небольшое улучшение самоорганизации (gate_std +8%).

### 3.5 VSA Vector Memory

**Что делает.** Ведёт пер-слойную векторную память через рекуррентность с гейтами:

```
mem[t] = decay[t] · mem[t-1] + i_gate[t] · h[t]
```

**Гейты:**

```python
decay[t]  = sigmoid(h[t] · w_d + b_d)   ∈ (0, 1) — насколько забыть
i_gate[t] = sigmoid(h[t] · w_i + b_i)   ∈ (0, 1) — насколько записать
```

**Чтение из памяти:**

```python
mem_read = mem_all * w_q + mem_leaf * w_q_leaf + mem_all * w_q_ctx
```

Три readout-вектора: `w_q` (основной, полная память), `w_q_leaf` (внутри-чанковое чтение без кросс-чанк контекста), `w_q_ctx` (кросс-чанковый контекст). Dual readout даёт модели выбор между локальной и глобальной информацией.

**Момент первого порядка (коррекция среднего):**

```python
mu_all = prefix_scan(decay, h * i_gate * w_k_mu)    # (B, L, D)
mu_read = mu_all * w_q_mu                            # (B, L, D)
mem_read = mem_read + mu_read * w_mu_mem             # (B, L, D)
```

Момент первого порядка даёт памяти информацию не только о "текущем среднем" (центроид), но и о дисперсии.

**2-level hierarchical prefix scan (CHUNK=32).**

Рекуррентность `mem[t] = a[t] · mem[t-1] + b[t]` вычисляется через два уровня:

**Level 1 — параллельные сканы чанков:** каждый чанк из 32 токенов сканируется от нулевого начального состояния. Результат — внутри-чанковая память `intra_k` и конечное состояние `final_k` для каждого чанка.

**Level 2 — последовательный cross-chunk scan:** состояния чанков `final_k` сканируются последовательно (32 шага для L=1024), комбинируя чанки через `state_{k+1} = inter_decay_k · state_k + final_k`.

**Итоговый результат:** `mem[t] = cross_k · cum_decay_k[t] + intra_k[t]` — экспоненциально затухающий кросс-чанковый контекст, добавленный к внутри-чанковой памяти.

**Зачем два уровня?** Параллельные сканы чанков дают O(L) на level 1 + O(L/CHUNK) на level 2. При L=128 это 4 чанка × 32 + 4 = 132 шага — на 3% больше, чем последовательный scan. При L=8192: 256 × 32 + 256 = 8448 — на 3% больше. Но численная стабильность: `exp(-log_cum)` считается только на 32 шагах, где `log_cum[32] ∈ [-20, 0]` даже при τ=8, без переполнения fp16.

**fp32 guard:** весь VSA scan форсирован в float32 (`decay.float()`, `input_vec.float()`) для численной стабильности под AMP (mixed precision). Результат кастуется обратно в исходный dtype.

**Surprisal-gated write (i_gate):**

```python
i_gate = softplus(h·w_i + b_i + γ·‖ê‖₂)
```

где `ê = (hp - pred_k) / ||hp||` — нормированная ошибка предсказания VSA-памяти. Токены с большой ошибкой (высокий surprisal) записываются в память сильнее. γ_l — обучаемый per-layer скаляр, инициализированный по τ: `γ_init = γ_max · σ((ln τ_l — ln 32) / 1.0)`.

**Калибровка decay (τ) — теперь per-channel, до τ≈160K.**

Эффективный timescale τ = -1 / ln(sigmoid(b_d)). b_d — D-мерный вектор, каждый канал учит свой timescale:

| b_d | sigmoid(b_d) | τ | Ретенция через 128 шагов | Ретенция через 100K шагов |
|---|---|---|---|---|
| 0.0 | 0.50 | 1.4 | 0.5¹²⁸ ≈ 0 | 0 |
| 2.0 | 0.88 | 7.8 | ≈ 1e-7 | 0 |
| 3.0 | 0.95 | 20 | 0.1% | 0 |
| **5.0** | **0.9933** | **150** | **42%** | 0 |
| **8.0** | **0.9997** | **3,000** | **96%** | ≈ 0 |
| **10.0** | **0.99995** | **22,000** | **99.4%** | 1.1% |
| **12.0** | **0.999994** | **163,000** | **99.9%** | **54%** |

При b_d=12.0 (τ≈163K): после 100K шагов 54% информации сохранено — контекст в сотни тысяч токенов.

**Per-channel adaptation.** b_d — `(D,)` вектор (4096 скаляров), каждый со своим LR=0.1×base. AdaptiveController задаёт exploration-based baseline, градиент дифференцирует per-channel. Каналы специализируются: одни — короткая память (τ≈10, быстрое забывание шума), другие — длинная (τ≈160K, permanent storage).

**Калибровка i_gate.**

`b_i = -3.0` → i_gate ≈ sigmoid(-3.0) = 0.047. На ините только 4.7% h[t] записывается в память за шаг. **Noise_scale** (5% на ините, decay с differentiation) добавляет мультипликативный шум на i_gate при тренировке: `i_gate *= (1 + ε·noise_scale)`, ε ~ N(0,1). Это исследовательский шум — при diff=0 шум 5%, при diff=1 шум 0.1%. При τ≈150 общая память после 128 шагов:

```
mem[128] ≈ Σ(decay^t · i_gate · h[128-t]) ≈ i_gate · h · τ ≈ 0.047 · 7.0 ≈ 0.33·h
```

То есть mem_all имеет std ≈ 5-6, что здорово (не эксплодирует, не затухает). Для сравнения: если бы i_gate был exp(...) вместо sigmoid, mem_all бы имел std ≈ 640 на ините → NaN.

### 3.6 Grouped Cognitive Mirror

**Что делает.** Ансамбль из **G=32 независимых экспертов-зеркал**. Каждый эксперт живёт в своём d=128-мерном подпространстве (D=4096/32) и вычисляет **ограниченную, пер-дименсионную коррекцию** на основе **4 сигналов самосогласованности** в своём K-space (k_l∈{4,8,8,16} по третям глубины).

**Зачем группа, а не одно зеркало?** Одно Cognitive Mirror не могло дифференцироваться — var(log_scale) ≈ 0.000048. Tanh-насыщение убивает градиент. 32 эксперта с d=128, k_l дают 32× больше независимых градиентных сигналов.

**Четыре сигнала коррекции (EMA-нормированные, learnable softmax-веса):**

| Сигнал | Формула | Что измеряет |
|---|---|---|
| **temp** — темпоральный | `(hp - mc) * w_temp` | Отклонение от среднего VSA-памяти |
| **pred** — предиктивный | `(hp - alpha_g * hp_prev) / ||hp||` | Ошибка предсказания K-space (t-1 → t), нормированная |
| **smooth** — сглаживающий | `hp - causal_conv1d(hp)` | Локальная негладкость K-space |
| **sym** — симметричный | `(hp·u)·(hp_prev·v)` | Асимметрия эволюции во времени |

Каждый сигнал перед взвешиванием нормируется per-экспертной EMA-нормой (RMS, τ≈1000). 4 learnable softmax-веса `w_i` определяют вклад каждого сигнала. `w_pred_scale` заменён на EMA-нормировку (оставлен как `w_pred_scale_legacy` для обратной совместимости чекпоинтов).

**φ-координата и геометрическая init:** каждый эксперт имеет φ = log(1+l)/log(n_layers) — единую логарифмическую координату глубины. Per-expert скаляры инициализируются геометрически: skip_alpha (L0≈17, L31≈0.10), log_dvar_mod_scale (L0≈—0.81, L31≈—2.30) с ρ=0.6^layer_idx. Это воспроизводит эмпирический градиентный ландшафт глубины.

**Все 4 сигнала суммируются по всем k=32 размерностям** (без lo/hi split). pred_error даёт полный градиент всем k dims, что в абляции даёт +24% движения alpha.

**Predictive Mirror: W_pred → alpha (scalar per expert)**

```python
# Было: W_pred (G, k, k) = 16K params — градиент 0.008/шаг, не учился
pred_k = einsum('blgk,gkk->blgk', hp_prev, W_pred)

# Стало: alpha (G,) = 32 scalars — градиент 3.1× быстрее
# плюс EMA-нормировка (signal_norm_ema) вместо w_pred_scale
pred_k = alpha_g * hp_prev   # temporal scaling: hp_t = alpha_g * hp_{t-1}
pred_error = (hp - pred_k) / ||hp||   # нормированная, без w_pred_scale
```

W_pred (G×k×k = 32×32×32 = 32K) заменён на скаляр alpha (G = 32 параметра). Per-param градиент в 1024× сильнее. Temporal scaling (|1-alpha|) учится за первые сотни шагов (0.99 → 0.95 на D=896 за 100 шагов).

**Важно: alpha > 1 = amplification, не decay.** Практически все 1024 эксперта (99%) после тренировки имеют alpha > 1.0 (среднее 1.012). Это означает, что модель ожидает **роста** K-space, а не затухания: `pred_k = 1.012 * hp_prev`. Temporal scaling — амплификация предыдущего состояния для создания предсказуемой ошибки `pred_error`, по которой эксперты специализируются. Лишь 11/1024 экспертов ниже 0.99, и почти все они на слое 0 (фильтр входа).

**Архитектура каждого эксперта:**
```python
d = 128          # размер подпространства
k = 32           # размер K-space

hp = h_g @ W_proj[g]            # (B, L, 128) → (B, L, 32)
hp_prev = pad(hp[:, :-1])       # сдвиг на 1

# Четыре сигнала → полная сумма по k
temp = (hp - mc) * w_temp
pred_error = (hp - alpha_g * hp_prev) * w_pred_scale  # alpha — скаляр!
smooth = hp - conv1d_depthwise(hp)
sym = (hp * w_u) * (hp_prev * w_v)

delta = temp + pred_error + smooth + sym + tanh_bias  # (B, L, 32), все k dims

linear = delta @ W_out[g]                   # (B, L, 32) → (B, L, 128)
mirror_g = tanh(linear) + α_g * linear      # per-expert skip connection

**tie_mirror_proj (по умолчанию True):** `W_out[g] = W_proj[g]^T` — автоэнкодерное ограничение per-expert. `W_out` — буфер, синхронизированный с `W_proj.permute(0,2,1)`. Экономия `G × k × d = 32×32×128 = 131K` параметров на слой, ~4.2M всего. Почти без потери loss.
mirror_g = mirror_g * exp(log_scale[g])     # per-dim amplitude
```

**K-Space Gate (per-token, per-expert):**

Gate от |pred_error|: gate открыт когда предсказание плохое, закрыт когда хорошее.

```python
gate_signal = torch.abs(pred_error)                    # (B, L, G, 32)
gate_logit = einsum('blgk,gk->blg', gate_signal, w_gate) + b_gate
expert_gate = sigmoid(gate_logit)                      # (B, L, G) — per-token!
mirror = mirror * expert_gate.unsqueeze(-1)
```

**alpha в gate_no_decay group** — weight_decay=0, как было с W_pred. После тренировки alpha живёт в [0.95, 1.03] (среднее 1.012, max 1.031, min 0.518 на L0).

**Root cause подтверждён (D=256, L=2, G=4, k=4, 500 steps):**
- Approach B (scalar α) — 3.1× быстрее baseline |I-diff| (0.0183 vs 0.0026)
- Approach A (remove split) — +24%
- Approach C (InfoNCE) — хуже baseline

**На случайных данных alpha → 1.0** (нет временной структуры).
**На структурированных данных alpha → 1.012** (temporal amplification: 1011/1024 экспертов > 1.0).

**alpha учится, W_pred — нет.** Десятки тысячелетий шагов (микро-эксперименты: 15000 шагов, D=896, L=8) W_pred не выходил из identity. Alpha начинает двигаться с первых сотен шагов.

### 3.7 Grouped MLP

**Что делает.** Основной потребитель параметров (88.9%). Заменяет стандартный MLP (D→D→D) на групповую структуру: D=4096 делится на 32 группы по 128, каждая с 4× внутренним расширением.

```
# Вход: h (B, L, D=4096)

# RMS Norm (как в flat MLP)
h = rms_norm(h, (D=4096,), norm_w)

# Разбивка на группы
h = h.reshape(B, L, G=32, d=128)

# Пер-групповой MLP (независимый!)
for g in range(G):
    h_up = silu(h_g @ W_up[g])    # 128 → 512 (4× расширение)
    h_down = h_up @ W_down[g]     # 512 → 128

# Обратная разбивка
h = h.reshape(B, L, D=4096)
```

Реализуется как один einsum:

```python
h = F.silu(torch.einsum('blgd,gdf->blgf', h, W_up))   # (B, L, 32, 896)
h = torch.einsum('blgf,gfd->blgd', h, W_down)           # (B, L, 32, 128)
h = h.reshape(B, L, D)
```

**Сравнение конфигураций:**

| Конфигурация | Параметров | Промежуточных нейронов | Эффективный ранг (L23) |
|---|---|---|---|
| Flat D→D→D (4096→4096→4096) | 33,554,432 | 4096 | 8-12 (0.3%) |
| **Grouped G=32, expand=8** | **6,422,528** | **28,672** | **70-90 (63-80%)** |

Grouped MLP даёт **в 8× больше промежуточных нейронов** при том же числе параметров. В flat MLP число промежуточных нейронов = D (количество не растёт с расширением — это не bottleneck, а равномерный MLP). В Grouped MLP: 32 группы × 896 = 28,672 нейрона — в 8× больше.

**Эмпирика: почему группы работают лучше?**

На шаге 15000 (D=3584, G=32, d=112):

| Слой | Flat MLP (eff_rank/D) | Grouped G=32 (eff_rank/d) |
|---|---|---|
| L0 | 320/3584 (9%) | 96/112 (86%) |
| L5 | 180/3584 (5%) | 91/112 (81%) |
| L10 | 95/3584 (2.7%) | 85/112 (76%) |
| L15 | 42/3584 (1.2%) | 74/112 (66%) |
| L20 | 18/3584 (0.5%) | 52/112 (46%) |
| L23 | 8/3584 (0.2%) | 35/112 (31%) |

Grouped MLP держит эффективный ранг на уровне 31-86% от d даже на глубоких слоях, где flat MLP коллапсирует до 0.2%.

**Почему группы не мешают друг другу?** Потому что каждая группа оперирует в своём d-мерном подпространстве (d=128). Группа 17 может специализироваться на синтаксисе, группа 23 на семантике, группа 5 на пунктуации — они не конкурируют за параметры.

### 3.8 DCT Spectral

**Что делает.** Применяет обученное пер-дименсионное частотное масштабирование к h через дискретное косинусное преобразование (DCT-II).

```python
V = dct_basis(D=4096)                    # DCT-II матрица (4096×4096), ортогональная
h_dct = h @ V.T                          # (B, L, 4096) → DCT коэффициенты
h = h + (h_dct * lambda_k) @ V           # масштабирование + обратное DCT
```

**lambda_k** — обученный D-мерный вектор, инициализированный линейно:

```python
lambda_k[layer] = 0.5 + layer_idx / (n_layers - 1)
```

Для 32 слоёв: L0 = 0.5 (все частоты подавлены вдвое), L16 = 0.984 (почти нейтрально), L31 = 1.5 (все частоты усилены в 1.5×).

**Что lambda_k учит?** Каждый из D=4096 DCT-компонентов имеет свой коэффициент масштабирования. Модель может:
- Усилить высокие частоты (детали, редкие паттерны)
- Подавить низкие частоты (шум, общий фон)
- Сделать частотно-избирательное усиление (одни частоты поднять, другие опустить)

После тренировки lambda_k на глубоких слоях (L26-L31) имеет per-dim std 0.01-0.05 — модель дифференцирует частоты, но не агрессивно. На ранних слоях (L0-L5) lambda_k практически константа — все частоты обрабатываются одинаково.

**Гипотеза специализации слоёв:**

| Слои | lambda_k | Режим | Предполагаемая роль |
|---|---|---|---|
| L0-L6 | 0.50-0.69 | Все частоты подавлены | Контекстная интеграция — построение стабильного представления |
| L7-L16 | 0.71-1.02 | Почти единица | Синтаксис — все частоты одинаково важны |
| L17-L24 | 1.03-1.27 | Умеренное усиление | Лексика — начинает подчёркивать паттерны |
| L25-L30 | 1.31-1.47 | Сильное усиление | Детали — дискриминативные признаки |
| L31 | 1.50 | Максимальное усиление | Сжатие для LM head |

### 3.9 LM Head (Partitioned Readout)

**Что делает.** Проецирует финальное h (D=4096) в логиты для 50000 токенов через **те же 32 сегмента**, что и эмбеддинг.

```python
h_final = rms_norm(h, (D,), final_norm_w)     # финальная RMS нормировка
# Разбиение на 32 сегмента × 128
for k in range(32):
    h_k = h[:, :, k*128:(k+1)*128]
    score_k = ⟨h_k, r_k⟩                        # скаляр — согласие сегмента и readout
logit_v = Σ_k z_{vk} · score_k                  # только активные для v биты
```

**Ключевое свойство:** голова не имеет cross-talk. Бит k читает ТОЛЬКО сегмент k финального h. Никакого `⟨h₃, r₇⟩` — каждый сегмент независимо голосует за свой бит. 

Это принципиально меняет динамику обучения:
- Информация должна быть **организована по сегментам** к последнему слою
- Bind (K=64), mirror (32 группы) и MLP (32 группы) — единственные каналы кросс-сегментного обмена
- Модель не может «схитрить» и перемешать всё в голове

**Сравнение:**

| Подход | Параметров | Cross-talk | Сегментная организация |
|---|---|---|---|
| Learned embedding | 4096×50000 = 204.8M | полный | нет |
| Linear Zeckendorf | 4096×23 + 23×50000 ≈ 1.17M | полный (D→K) | нет |
| **Partitioned Head** | **32×128 = 4096** | **нет** | **принудительная** |

Параметров: 4096 против 1.17M — фактор 286× экономии. Но главное не экономия, а структурное принуждение: h на последнем слое обязан быть организован по 32 сегментам, иначе head не может прочитать информацию.

**Как градиент течёт:** `∂L/∂logit_v = error_v`. `∂L/∂r_k = Σ_{v: z_k=1} z_{vk} · error_v · h_k`. Каждый r_k получает градиент только от токенов с z_k=1 — ~9375 токенов из 50000. Остальные 40625 не шумят в r_k. Это **градиентная группировка** — каждый readout учится от своего непересекающегося подмножества словаря.

**Сегменты и residual:** через 24 слоя gradient от r_k (только z_k=1) распространяется обратно ко всем компонентам. Bind, mirror и MLP вынуждены научиться направлять информацию от любых сегментов к тем, которые активирует целевой токен — это самодостаточный механизм организации скрытого пространства.

### 3.10 Adaptive Controller

**Что делает.** Полностью автоматическая подстройка всех гиперпараметров VSA-памяти. Никаких ручных learning rate для гейтов, никакого grid search. Всё управляется двумя сигналами из Cognitive Mirror.

**Два фундаментальных сигнала:**

```python
exploration = min(1, |mirror| / 0.3)
    # Насколько сильно зеркало корректирует выход.
    # Высокое → модель активно подстраивается, нужна агрессивная память.
    # Низкое → модель стабильна, нужна консервативная память.

differentiation = min(1, var(log_scale) / 0.1)
    # Насколько зеркало специализировалось пер-дименсионно.
    # Высокое → зеркало знает, какие размерности подавлять/усилять.
    # Низкое → зеркало ещё исследует.
```

**Пять адаптируемых параметров:**

| Параметр | Диапазон | От чего зависит | Формула | Config поля |
|---|---|---|---|---|
| b_d (bias decay) | [2.0, b_d_max] → τ ≈ [8, 163K] (per-channel) | exploration + layer + gradient | `b_d_max - expl·(b_d_max - (2.0+3.0·layer_frac))` | lerp к target, cap=b_d_max=12.0, per-channel (D,) |
| b_i (bias write gate) | [-3.0, -1.5] → i_gate ≈ [0.047, 0.27] | exploration | `-3.0 + expl·1.5` | fill_, сужение, i_gate≤0.27 |
| w_mem2v_scale | [min, max] → [0.5, 1.0] | differentiation | `max - diff·(max-min)` | `w_mem2v_scale_min/max` |
| EMA α (global state) | [min, max] → [0.90, 0.99] | differentiation | `min + diff·(max-min)` | `ema_alpha_min/max` |
| EMA α (δ_var gate) | [min, max] → [0.80, 0.99] | differentiation | `min + diff·(max-min)` | `delta_var_ema_min/max` |
| Noise scale (гейты) | [min, max] → [0.001, 0.05] | differentiation | `max - diff·(max-min)` | `noise_scale_min/max` |

**Ключевые изменения:** b_d теперь **пер-слойный и пер-канальный**: L0(τ≈8) → L31(τ≈`vsa_b_d_max` на expl=0). Каждый из D=4096 каналов учит свой timescale через градиент (LR=0.1×base), AdaptiveController задаёт exploration-based soft target через lerp. Верхняя граница b_d: `vsa_b_d_max=12.0` (τ≈163K) — позволяет единичным каналам хранить информацию на 100K+ шагов. i_gate не нормирован (безопасность через градиент + lerp-регуляризацию).

b_i сужен: `-3.0 + expl·1.5` (было `-3.0 + expl·3.0`). При expl=0: i_gate=0.047, при expl=1: i_gate=0.27. Старая формула давала i_gate=0.5 при expl=1 → перегрузка VSA-памяти (||mem||² ~ 26, целевой ~5-6). Новая — i_gate ≤ 0.27, память в норме.

**Gate LR:** gate_lr_mult=5.0. W_pred — в `gate_no_decay` (weight_decay=0, LR×5). Причина: per-param градиент W_pred в 42× меньше чем у pred_scale, weight decay (0.01) пересиливал градиент (3:1). WD=0 — essential fix.

**Интуиция:**

- **exploration высокий** (зеркало делает большие коррекции) → Память должна быть короткой (низкий τ/high b_d → decay быстрый) и писать агрессивно (high b_i → много информации). Почему? Потому что зеркало говорит "модель активно учится, старые данные в памяти устаревают быстро".

- **differentiation высокая** (зеркало специализировалось) → Память должна уступить зеркалу (низкий w_mem2v_scale → меньше памяти в выходе) и обновлять свою self-модель медленно (высокий EMA α). Почему? Потому что зеркало уже знает, какие размерности важны, и не нуждается в частых пересмотрах.

**Per-layer Centroid EMA (global_state):**

global_state расширен до `(n_layers, 1, D)` — каждый слой ведёт свою self-модель. Скорость EMA per-layer:
```python
c_ema = (1 — 0.976) · 149 ≈ 3.576    # калибровка по L31
α_l = 1 — c_ema / τ_l                # L0≈0.52 (τ≈7), L31≈0.976 (τ≈149)
```
Глубокие слои (большое τ) обновляют центроид быстро, поверхностные — медленно.

**Интеграция в forward:**

```python
# WideBindStack.forward():
expl, diff = AdaptiveController.stats(layers,      # thresholds из config
    expl_thresh=cfg.exploration_threshold,
    diff_thresh=cfg.differentiation_threshold)
n = len(layers)
for i, layer in enumerate(layers):
    layer_frac = i / max(n - 1, 1)
    b_i_val = -3.0 + expl * 1.5
    b_d_val = (2.0 + 3.0 * layer_frac) + expl * (5.0 - (2.0 + 3.0 * layer_frac))
    layer.b_i.fill_(b_i_val)                         # ! fill_ — не grad!
    layer.b_d.fill_(b_d_val)
mem2v_scale = AdaptiveController.w_mem2v_scale(layers,   # range из config
    min_val=cfg.w_mem2v_scale_min, max_val=cfg.w_mem2v_scale_max)
ema_alpha = AdaptiveController.ema_alpha(layers,         # range из config
    min_val=cfg.ema_alpha_min, max_val=cfg.ema_alpha_max)

# WideBindBlock.forward():
enhanced = bind_out + mem_read * w_mem2v * mem2v_scale + mirror
```

Обратите внимание: `.fill_()` устанавливает bias напрямую, минуя градиент. b_i и b_d исключены из optimiser parameter groups (`param_groups()` не включает их). Это предотвращает конфликт между адаптивным контролем и градиентными обновлениями.

W_pred — в `gate_no_decay` (weight_decay=0, LR×5). Per-param градиент W_pred (усредняется через einsum по B×L) в 42× меньше чем у pred_scale (поэлементный). С weight_decay=0.01 градиент не мог преодолеть wd. WD=0 — ключевое исправление catch-22. w_pred_scale в `gate_no_decay` (WD=0, LR×5) — наравне с W_pred, так как его градиент также ослаблен через цепочку |pred_error| → gate → mirror.

### 3.11 MirrorLRScheduler

**Что делает.** Полностью **двунаправленная** модуляция learning rate на основе трёх сигналов Cognitive Mirror. LR может как расти (до 3× базового), так и падать (до 0.05×). **Нет forced cosine decay.**

**Три сигнала для LR:**

1. **var(log_scale)** — дисперсия пер-дименсионной амплитуды. Растёт от 0 (зеркало не дифференцировано) до ~0.1+ (специализировалось). Коэффициент: `var_mult = 1 + (var - target_var) / var_scale`.

2. **|1-alpha|** — насколько alpha отличается от 1.0. Высокое → эксперты имеют неединичный temporal scaling, нужно больше LR. Коэффициент: `alpha_mult = 1 + |1-alpha| / alpha_scale`.

3. **gate_var** — дисперсия bias-вектора гейтов по экспертам. Высокое → gate активно дифференцирует экспертов. Коэффициент: `gate_mult = 1 + gate_var / gate_scale`.

**Формула:**
```python
mag_factor = avg(|mirror|) / threshold
mult = max(lr_min_ratio, min(lr_max_ratio, var_mult × alpha_mult × gate_mult × mag_factor))
LR = mult × base_lr
```

**Почему bidirectional, а не только затухание?** Экспертная специализация — процесс chicken-and-egg: эксперты не специализируются без градиента, градиент слаб без специализации. Когда эксперты начинают дифференцироваться (var(ls)↑, |1-alpha|↑, gate_var↑), LR автоматически растёт, ускоряя процесс. Когда специализация насыщается — LR падает.

**Эмпирика (D=4096, L=32, 5000 шагов):**
- `var(ls)`: 0.0077 → 0.0080 (+4%) — умеренный рост
- `|1-alpha|`: 0.0077 → **0.0147** (+91%) — уверенная специализация
- `gate_var`: 0.0017 → **0.0910** (×53) — взрывная дифференциация гейтов
- `mult`: 0.88 → **3.0** (упирается в ceiling) — LR растёт вместе с gate_var

**Конфигурация:** Все параметры из `WideBindConfig`. Создаётся через `MirrorLRScheduler(model, optimizer, cfg=cfg)`.

```python
streams = [np.fromfile(f, dtype=np.uint16) for f in glob('token_stream_*_clean.bin')]
# Каждый поток: uint16 массив длиной 1.1B-1.8B токенов
```

Файлы `_clean.bin` — это очищенные версии (удалены перемежающиеся PAD токены из оригинальной записи).

**Форматы файлов:**

| Файл | Размер | Токенов | Источник |
|---|---|---|---|

uint16 → uint16: 2 байта на токен → 2.9 GB RAM. При обучении на Colab (T4 16GB) это влезает в RAM с запасом.

### Батчи

```python
def get_batch(stream, seq_len, batch_size, offset):
    needed = B * L + 1
    if offset + needed > len(stream):
        offset = 0  # конец потока → начало
    chunk = stream[offset:offset + needed]
    x = chunk[:B*L].view(B, L)      # вход
    y = chunk[1:B*L+1].view(B, L)   # target (сдвиг на 1)
    return x, y, offset + B*L
```

**B=2, L=64** — максимальный батч для Colab T4 при D=4096, L=32 (151M). 128 токенов на шаг.

### Оптимизатор

```python
optimizer = torch.optim.AdamW(
    param_groups, lr=3e-4, betas=(0.9, 0.95), weight_decay=0.01
)
```

**Parameter groups:**

- **Decay (weight_decay=0.01):** Все 2D параметры (матрицы — W_proj, W_out, W_up, W_down, conv.weight и т.д.)
- **No decay (weight_decay=0):** Все 1D параметры (biasы, gate-вектора, scale-вектора, norm-вектора)
- **Gate (LR×5, WD=0):** w_i, w_d, w_q, w_q_leaf, w_q_ctx, w_mem2v, w_k_mu, w_q_mu, w_mu_mem, w_u, w_v, w_gate, b_gate, mirror projs
- **Excluded:** b_i, b_d (адаптивный контроллер устанавливает их напрямую)

### Scheduler

MirrorLRScheduler — двунаправленная модуляция LR на основе var(ls), |1-alpha|, gate_var. Нет forced cosine decay.

### Soft EOS reset

При встрече токена EOS (id=2) в конце последовательности, VSA-состояние каждого слоя мягко сбрасывается: `state *= 0.1`. Это предотвращает загрязнение памяти меж-эпизодным контекстом, сохраняя при этом слабый сигнал о предыдущем эпизоде.

### Eval

```python
for stream in streams:
    offset = len(stream) // 4       # вторая четверть потока (hold-out)
    for _ in range(n_batches):
        x, y, offset = get_batch(...)
        h = model.embed_tokens(x)
        out, state = model(h, state)
        loss = model.compute_loss(out, y)
```

**10 warm-up батчей** — пропускается без замера, чтобы state + AdaptiveController стабилизировались.

**100 батчей на каждый из N потоков** — измерение.

### Loss

```python
logits = self.lm_head(h)
ce = F.cross_entropy(logits.reshape(-1, vocab), targets.reshape(-1), reduction='none')
mask = (targets != 0) & (targets != 2)  # PAD(0) и EOS(2) игнорируются
ce = ce * mask.float()
```

CE маскирует специальные токены (PAD=0, EOS=2) — loss считается только по валидным токенам. Surprisal-взвешивание усиливает градиент на информативных токенах:

```python
w = (ce / ce.mean()) ** γ  # w > 1 для токенов выше среднего loss
ce_loss = (ce * w).sum() / mask.sum()
```

| Компонент | Вес | Что делает |
|---|---|---|
| CE (PAD/EOS masked, surprisal-weighted) | 1.0 | `w_t = (CE_t / mean(CE))^γ` фокусируется на информативных токенах |
| pred_loss (K-space) | adaptive (0.05–1.0) | `MSE(pred_k, hp)` — учит alpha предсказывать K-space |
| gate_l1 | 0.001 | Спарисити гейтов экспертов |
| reinforce | 0.01 | `MSE(gate, usefulness)` — self-consistency RL |
| balance | 0.01 | `(logG — H_usage)/logG` — равномерное использование экспертов |
| diversity | 0.001 | `||cov(||group_out||) — I||²` — декорреляция MLP-групп |
| nuclear | 1e-5 | `||W_proj||_*` — стохастическая ядерная норма |
| orth | 1e-4 | `||Ŵ^T Ŵ — I||²` — ортогональность bind |
| w_m2v_hierarchy | 0–0.001 | L₂-регуляризация `w_mem2v` к таргету σ(ln τ_l), растёт с глубиной |
| branch_balance | 0–0.001 | `(logVar(Δconv) — logVar(Δbind))²` и т.д. — выравнивание ветвей |

Скалярный alpha (G,) не требует отдельного auxiliary loss — temporal scaling учится напрямую через градиент pred_error к alpha.

```python
for _ in range(max_new_tokens):
    ctx = tokens[-L:].unsqueeze(0)     # контекст не длиннее seq_len

    out, state = model(h, state)       # recurrent state переносится

    logits = logits / temperature
```

### FCF-CPR сжатие

Алгоритм крайне простой:

1. **Определение removable тензоров.** V_dct (D×D) и codes (vocab×K) — детерминированные, вычисляются заново из конфига.
2. **Удаление.** Скипаются полностью.
3. **Uniform 8-bit quantization.** Каждый оставшийся тензор независимо квантизуется в uint8: `index = round((value - min) / scale)`, где `scale = (max - min) / 255`. Хранится: uint8 тензор + 2 float (min, scale).
4. **Runtime-буферы** (`_last_gates`, `_last_h_pool`, `_prev_grad_norm`, `_last_magnitude`) — помечены `persistent=False`, не сохраняются в чекпоинт. Восстанавливаются на ините модели.

**Важно:** `decompress_sd` регенерирует `sparse_block_codes(vocab, K=cfg.code_dim, S=cfg.code_sparsity)` — не `zeckendorf_codes` из старой версии. При загрузке чекпоинта, сохранённого старой версией FCF-CPR, требуется патч (встроен в `colab.ipynb`, ячейка [4]).

**Ошибка:** MSE 1.7e-5 (наиболее пострадавший: W_proj, MSE 3.2e-5). Для val_loss: разница < 0.001.

## 9. Математическое приложение

### A. Доказательство ассоциативности prefix scan

Рекуррентность `mem[t] = a[t] · mem[t-1] + b[t]` можно записать как:

```
(mem[t], 1) = (a[t], b[t]) ⊗ (mem[t-1], 1)
            = (a[t] · mem[t-1] + b[t], 1)
```

Оператор ⊗ на парах (a, b) ассоциативен:

```
((a₁, b₁) ⊗ (a₂, b₂)) ⊗ (a₃, b₃)
= (a₁ · a₂, a₂ · b₁ + b₂) ⊗ (a₃, b₃)
= (a₁ · a₂ · a₃, a₃ · (a₂ · b₁ + b₂) + b₃)
= (a₁ · a₂ · a₃, a₂ · a₃ · b₁ + a₃ · b₂ + b₃)

(a₁, b₁) ⊗ ((a₂, b₂) ⊗ (a₃, b₃))
= (a₁, b₁) ⊗ (a₂ · a₃, a₃ · b₂ + b₃)
= (a₁ · a₂ · a₃, a₂ · a₃ · b₁ + a₃ · b₂ + b₃)

Same result. Ассоциативность выполняется → параллельный tree scan возможен.
```

### B. Почему bind работает (K=64) — градиентный анализ

```
forward: bind = (W_projᵀ h ⊙ w_u ⊙ w_v)ᵀ W_out
              = hᵀ W_proj · diag(w_u) · diag(w_v) · W_out
              = hᵀ M

где M = W_proj · diag(w_u) · diag(w_v) · W_out ∈ ℝᴰˣᴰ
```

Ранг M ≤ K (промежуточное представление K-мерно). При обратном распространении:

```
∂bind/∂h = Mᵀ  (полная D×D матрица, ранг K)
∂bind/∂W_proj = h ⊗ (diag(w_u) · diag(w_v) · W_out)  (внешнее произведение)
```

Важно: каждый элемент h влияет на все D элементов bind через K-мерный баттлнек. Якобиан M — не диагональный, в отличие от pure VSA.

### C. Effective Rank MLP

Effective rank (или stable rank) матрицы A ∈ ℝᵐˣⁿ с сингулярными числами σ₁ ≥ ... ≥ σ_min(m,n):

```
eff_rank(A) = (Σ σᵢ)² / Σ σᵢ²
```

Интерпретация: количество размерностей, несущих значимый сигнал.
- eff_rank = d → все размерности используются полностью
- eff_rank = 4 → только 4 из d размерностей несут сигнал, остальное шум

**Почему GroupedMLP имеет более высокий eff_rank чем Flat MLP?**

Flat MLP: `h_mlp = W_down · SiLU(W_up · h)`. rank(h_mlp) ≤ D. Внутреннее представление — D нейронов (размерность входа в SiLU).

GroupedMLP: 32 группы, каждая с d=128 → 512× внутреннее представление. Общее число SiLU-нейронов: 32 × 512 = 16,384 — в 4× больше.

SiUL применяется 8× чаще, каждая на своём подпространстве. Это даёт 8× более богатые нелинейные взаимодействия без роста параметров.

### D. DCT-II ортогональность

Базис DCT-II V удовлетворяет:

```
V[n,k] = α_n · sqrt(2/D) · cos(π · n · (k + 0.5) / D)
```
где α₀ = 1/√2 и α_n = 1 для n > 0.

Ортогональность: `V @ Vᵀ = I`. Доказательство:

```
Σ_k V[n₁,k] · V[n₂,k]
  = (2/D) · Σ_k cos(π·n₁·(k+0.5)/D) · cos(π·n₂·(k+0.5)/D) · α_n₁ · α_n₂
  = δ[n₁, n₂]
```

Для n₁ = n₂ = 0: `Σ_k V[0,k]² = (2/D) · D · (1/2) = 1`
Для n₁ = n₂ > 0: `Σ_k V[n,k]² = (2/D) · Σ_k cos²(...) = 1`
Для n₁ ≠ n₂: интеграл косинусного произведения равен нулю по DCT-решётке.

### F. Масштабирование: геометрическая согласованность

WideBind спроектирован для масштабирования без потери структурной целостности. **Инварианты, которые нельзя нарушать:**

```
d = 128          — фиксирован при любом G (d%k==0)
D = G · d        — размерность выводится из числа групп
K = G = L        — bind-каналы = группы = слои (всегда)
k = 32           — фиксирован (d/k = 128/32 = 4 — целое)
expand = 4       — фиксирован (оптимален для d=128)
S = max(4, round(K · 0.1875))  — ~18.75% активных бит
```

**Скелет масштабирования (d=128, k=32, expand=4, S=6, tied projections):**

| G (=K=L) | D | Bind | MLP | Mirror | Total | Tok/param (8B) | Hardware |
|---|---|---|---|---|---|---|---|
| 8 | 1024 | 8K | 1.0M | 8K | **18M** | 444 | CPU/любой |
| 16 | 2048 | 33K | 2.1M | 33K | **40M** | 200 | MX550 |
| **32** | **4096** | **131K** | **4.2M** | **131K** | **151M** | **53** | **T4** |
| 48 | 6144 | 295K | 6.3M | 295K | **343M** | 23 | RTX 3090 |
| 64 | 8192 | 524K | 8.4M | 524K | **612M** | 13 | RTX 3090 |
| 96 | 12288 | 1.2M | 12.6M | 1.2M | **1.4B** | 5.7 | A100 |
| 128 | 16384 | 2.1M | 16.8M | 2.1M | **2.6B** | 3.1 | 2×A100 |
| 256 | 32768 | 8.4M | 33.6M | 8.4M | **11.2B** | 0.71 | H100 cluster |

Bind/Mirror колонки: tied projections (W_out = W_proj^T), per-layer.

**Что происходит при каждом удвоении G:**

| Компонент | Масштаб | Природа |
|---|---|---|
| D | ×2 | D = G·d |
| Bind | ×4 | O(D·K) = O(G²·d²) — квадратично |
| MLP | ×2 | O(G·d²·expand) — линейно по G |
| Mirror | ×2 | O(G·d·k) — линейно |
| Conv+VSA+Spectral | ×2 | O(D) — линейно |
| **Total** | **~×4** | **доминирует Bind на больших G** |

**Ключевой момент:** при G>64 Bind начинает догонять MLP:
```
G=32:  bind/MLP = 3.6%   (MLP рулит)
G=64:  bind/MLP = 7.2%
G=128: bind/MLP = 14.3%
G=256: bind/MLP = 28.6%  (Bind значим)
```

**Sequence length:** не нарушает геометрию. VSA — O(L log L) пер-слойно. Принципиальное преимущество WideBind: L не влияет на веса, только на активации. `seq_len` можно менять в любой момент без перетренировки:

| seq_len | VRAM (inf, fp16) | tok/s (MX550) | Замечание |
|---|---|---|---|
| 128 | 0.55 GB | 11 | Текущий |
| 512 | 0.65 GB | 8 | VSA scan ×4.5 |
| 2048 | 0.95 GB | 4 | Граница MX550 |
| 8192 | 1.80 GB | 1 | MX550 предел |

**code_sparsity (S):** S = 6 при K=32 даёт 6/32 = 18.75% активных бит. При масштабировании K, S должен расти пропорционально:
```
S(K) = max(4, round(K · 0.18))  — держать ~18% активность
K=32  → S=6
K=64  → S=12
K=128 → S=23
K=256 → S=46
```

**Процедура масштабирования (дорожная карта):**

```python
# 1. Выбрать G → вычисляем все производные размеры
G  = 64                               # цель
d  = 128                              # фиксирован
D  = G * d                            # 8192
bind_K = 2 * G                        # 128
k  = 32                               # k_l staircase: 4/8/8/16
S  = round(32 * 0.18)                 # 6
L  = G                                # слои = группы

# 2. Создать WideBindConfig — все числа самосогласованы
cfg = WideBindConfig(D=D, n_layers=G, bind_K=bind_K,
                     mlp_groups=G, code_dim=32,
                     code_sparsity=S, mirror_k=k)

# 3. Инициализировать модель
model = WideBindStack(cfg)  # все выровнено 1:1

# 4. Загрузить чекпоинт меньшей модели (zero-shot transfer)
#    W_proj: (D_old, K_old) → (D_new, K_new) — interpolate
#    W_up:   (G_old, d, 8d) → (G_new, d, 8d) — pad new groups as zeros
```

**Перенос весов с G_old на G_new:**

| Компонент | Стратегия переноса |
|---|---|
| Bind (W_proj, W_out) | Интерполяция: D_old×K_old → D_new×K_new |
| MLP (W_up, W_down) | Новые группы = zero-init (не интерполировать) |
| Mirror (W_proj, W_out, W_pred) | Новые эксперты = копия ближайшего |
| w_gate | Zero-init для новых экспертов |
| VSA gates (w_d, w_i, ...) | Интерполяция D_old → D_new |
| PartitionedHead readout | Новые сегменты = zero-init |

**Итог:** масштабирование WideBind — это выбор одного числа G (число групп), из которого все остальные размеры выводятся автоматически. Геометрия зафиксирована: `D = G·d = G·128`, `L = G`, `bind_K = 2·G`.

## 10. FAQ

**Почему модель генерирует только PAD?** Модель на 6000 шагов (~151M params) — это ~0.005 токена на параметр. Типичные LM требуют >1 токена на параметр для осмысленной генерации. Текущая val_loss=1.90 — модель учится, но генерация потребует на порядок больше шагов.

**Почему n_layers=32, а не 24?** D/L = 4096/32 = 128 = d (целое). L=24 давал бы D/L = 170.67 — архитектурный «остаток». L=32 совпадает с числом групп G=32 и числом экспертов зеркала. Каждый слой маппится 1:1 на группу/эксперта.

**Почему bind_K=64, а не 32?** bind_K=64 даёт D/K = 64, эффективный ранг 38-50/64 (59-78%). K=64 выбран для увеличения ёмкости скрещивания без квадратичного роста параметров. С tied проекциями добавка всего +262K параметров относительно K=32.

**Почему D=4096, а не 8192?** D=4096 выбран как стандартная степень двойки для ML-моделей, при которой G=32 даёт d=128 (целое). D=8192 дал бы d=256 при G=32, ~1.1B параметров — OOM на T4. D=4096 с tied projections даёт 151M — влезает на T4 16GB с B=2, seq_len=64.

**Почему не добавить MoE или sparse mixture?** MoE требует routing + auxiliary loss. GroupedMLP — "бедного человека MoE": 32 группы всегда активны, но каждая специализируется на своём подпространстве.

**Можно ли заменить sparse block codes на learned embedding?** Технически — да. Практически — 179M лишних параметров. Если когда-нибудь будет GPU с 24GB+, можно попробовать.

**Как масштабировать архитектуру?** Весь скелет масштабирования — в разделе [9.F](#f-масштабирование-геометрическая-согласованность). Выбираешь G, все размеры выводятся автоматически: D = G·128, L = G, bind_K = 2·G. Геометрия не меняется.

**Зачем нужен train.bat?** Для запуска тренировки двойным кликом на Windows без открытия терминала: `python scripts/train.py --data-dir C:\Users\...\fcp`.
