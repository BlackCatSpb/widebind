# WideBind для занятых

**Это языковая модель. Она не transformer. В ней нет attention, нет softmax, нет KV-cache, нет QKV-проекций. Всё иначе.**

---

## Зачем вообще это делать?

У transformer три врождённых дефекта, которые нельзя исправить апгрейдами:

1. **O(L²) attention** — каждый токен смотрит на каждый. На длинных текстах это квадратичный взрыв.
2. **KV-cache растёт с длиной** — генерация длинных текстов упирается не в вычисления, а в память.
3. **Softmax bottleneck** — веса внимания обязаны быть положительными и давать сумму 1. Модель не может сказать «этот токен НЕ важен, вес -1».

WideBind решает все три проблемы заменой механизма, а не оптимизацией.

---

## Три идеи, на которых всё держится

### 1. Память не матрица, а вектор

Transformer хранит матрицу K×V (все прошлые токены). WideBind хранит один D-мерный вектор — **суперпозицию**. Новый токен вписывается в неё с регулируемой силой (write gate), старый затухает (decay).

Это выглядит как рекуррентная сеть, но вычисляется параллельно — через prefix scan.

**Результат:** состояние ~16 KB на слой (D=4096 в fp16). Генерировать можно миллион токенов — объём памяти не растёт.

### 2. Скрещивание размерностей через биллинейность, не через weighted sum

Transformer вычисляет QK^T (каждый токен × каждый). WideBind проецирует D→32, перемножает две проекции поэлементно (u·v), проецирует обратно 32→D.

Это не weighted sum — это **биллинейное скрещивание**. Размерности «общаются» через общее узкое горлышко K=32, а не попарно.

### 3. Вместо LayerNorm — Cognitive Mirror (32 эксперта с alpha)

LayerNorm просто нормализует вход. Cognitive Mirror делает иное: учится **корректировать представления** каждого слоя через K-space (32 dims на эксперта).

У каждого из 32 экспертов четыре сигнала коррекции, суммируемых по всем размерностям K-space:
- **temp** — отклонение от VSA-памяти
- **pred** — ошибка предсказания: `pred_error = hp - alpha_g * hp_prev` (скалярный alpha на эксперта)
- **smooth** — локальная негладкость
- **sym** — билинейная асимметрия t-1→t

Ключевое: `alpha` (G скаляров) заменил `W_pred` (G×k×k = 32K параметров) — temporal dampening учится за первые сотни шагов, а не стоит на месте 10000 шагов как было с W_pred.

## Дальше — проще

- **Эмбеддинг:** не learned embedding table, а sparse block codes (ровно 6 бит из 32 на токен, комбинаторика C(32,6) ≥ 50000 словаря). Фиксированное разреженное представление.
- **Bind:** D → K=32 → bilinear → D. Сжатие, скрещивание, разжатие.
- **VSA Memory:** prefix scan над векторами с learnable decay и write gate.
- **MLP:** 32 параллельных маленьких MLP вместо одного большого.
- **DCT Spectral:** адаптивная фильтрация в частотной области (DCT-II) — без learnable позиционных кодировок.
- **AdaptiveController:** мета-контроллер, который в каждый forward вычисляет 5 гиперпараметров памяти из двух сигналов — «как сильно зеркало корректирует» и «насколько зеркало специализировалось».

---

## Архитектура одной строкой

```
token → Sparse Embed (32×112) → [Block × 32] → RMS Norm → LM Head
                                 │
                           Block: Pre-LN → Conv → Bind → VSA Memory + Mirror + DCT + MLP
```

Все 32 слоя, 32 MLP-группы, 32 эксперта зеркала, 32 bind-канала — выровнены 1:1. D = 32 × 128.

| Размер чекпоинта | ~293 MB |
| Temporal gate | Scalar alpha (G,), не W_pred |

---

## Чем не является

- **Не RNN** — параллелизуется через prefix scan
- **Не State Space Model** — нет диагонализации, нет HiPPO/S4/S5
- **Не mixture of experts** — все эксперты зеркала работают всегда, gate только модулирует выход
- **Не линейная модель** — есть tanh, биллинейность, conv, DCT
