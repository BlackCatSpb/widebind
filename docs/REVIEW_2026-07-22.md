# WideBind — Architecture Review (22 Jul 2026)

**161M params, D=4096, L=32, G=32, d=128, bind_K=64, BottleneckBind (shift).**

---

## 1. Strengths

- **Coherent geometry.** All dimensions derived from G: `D = G·d`, `L = G`, `bind_K = 64`, `k = 32`. No magic numbers.
- **Numerically stable VSA.** Chunked scan (CHUNK=32) + fp32 guard. NaN eradicated.
- **Zero softmax in body.** Sigmoid (gates) and tanh (mirror) only.
- **BottleneckBind with Fibonacci twist.** K=64 + golden-roll shifts raise output rank without parameter growth.
- **Tied projections.** autoencoder constraint on both bind and mirror. Saves ~400K params with near-zero loss impact.

---

## 2. Weaknesses

| Issue | Severity | Why |
|---|---|---|
| **GroupedMLP is plain SiLU, no gating** | 🟡 Medium | 88% params = `SiLU(x·W_up)·W_down`. SwiGLU gives ~2× nonlinearity for +15% params. |
| **VSA is purely additive** | 🟡 Medium | `mem = decay·mem + write·h`. No binding/unbinding, no complex VSA (FHRR), no multiplication. Pure linear recurrence. |
| **AdaptiveController is rule-based** | 🟡 Medium | 5 hyperparams derived by formula from expl/diff. Not learned. Replaceable by tiny MLP (64 params). |
| **No cross-layer memory** | 🔴 High | Each layer stores its own VSA vector. No skip-connections between layer memories. |
| **Data bottleneck** | 🔴 Critical | 6.3B tokens for 161M params. Rule of thumb (20 tokens/param) needs 3.2T. Current = 0.2% of that. |
| **Position encoding = Conv1d only** | 🟡 Medium | Conv1d (k=48) captures local position. No RoPE, no ALiBi, no long-range position signal. |
| **Single-device training** | 🔴 High | T4 16GB / MX550 2GB. No FSDP, no pipeline parallelism. 161M is the fp32 limit on T4. |

---

## 3. Proposed Improvements

### Priority 1 — High impact, low risk

| Improvement | Cost | Expected gain |
|---|---|---|
| **SwiGLU for GroupedMLP** | +15% params | ~0.05–0.1 nats val_loss |
| Enable `surprisal_weight` | 0 params | ~0.02–0.05 nats |

SwiGLU: replace single SiLU projection with gated variant.
```
h = SiLU(x·W_gate) ⊙ (x·W_up) @ W_down
```
Llama/Mistrel ablation: consistent -0.05 nats at iso-param.

### Priority 2 — Medium impact, medium risk

| Improvement | Cost | Expected gain |
|---|---|---|
| **Cross-layer VSA gate** | 0 params | Better depth-wise info flow |
| **Complex-valued VSA (FHRR)** | 0 params | Infinite retention without b_d=12 |
| **Learned AdaptiveController** | 64 params | Graceful curriculum instead of hard thresholds |

Cross-layer VSA:
```python
global_mem = mean(mem_l for l in top_k)
mem_l = decay·mem + write·h_l + β · global_mem
```
One line, zero params. Lets information flow between layer memories.

FHRR binding replaces multiplicative decay with rotational binding:
```
mem = mem ⊙ cos(θ·h) + write·h  # rotation, not attenuation
```
Memory does not decay — it transforms. ∞ timescale.

### Priority 3 — Speculative

| Improvement | Cost | Expected gain |
|---|---|---|
| **Positional bias in Conv (ALiBi-like)** | +D params | Weak position signal |
| **Multi-scale VSA (variable chunk size per layer)** | 0 params | Adaptive memory granularity |
| **Dropout on VSA write (stochastic i_gate)** | 0 params | Regularization |

## 4. Impact Assessment

### Info-theoretic bound

Current model: 161M params, 6.3B tokens. ~39 tokens/param. Typical llm needs ~1000× more.

All architectural improvements combined (SwiGLU + FHRR + cross-layer gate + adaptive controller + surprisal) are bounded by:
- **Upper estimate:** ~0.3 nats improvement on current data
- **True improvement:** ~0.05 nats on current data, up to ~0.5 nats at 60B+ tokens

**Implication:** data quantity dominates architecture past a relatively low threshold. The most impactful single change is scaling data, not architecture.

### Recommended sequence

```
Phase 1 (now):     SwiGLU + surprisal_weight → commit, validate on Mini
Phase 2 (next):    Cross-layer VSA gate → 1 line of code, easy
Phase 3 (scale):   Train on 3× more data (20B+ tokens) → biggest ROI
Phase 4 (explore): FHRR complex VSA or learned AdaptiveController
```
